## <a name="dynamic-service-plan"></a>Σχέδιο παροχής υπηρεσιών δυναμικής

Στο η δυναμική υπηρεσία σχεδιασμός, τις εφαρμογές σας συνάρτηση εκχωρούνται σε μια συνάρτηση παρουσία εφαρμογής. Εάν είναι απαραίτητο και σε άλλα σημεία θα προστεθεί δυναμικά.
Για τις παρουσίες μπορούν να εκτείνονται σε πολλούς πόρους υπολογιστικών, καθιστώντας στο έπακρο το διαθέσιμο Azure υποδομής. Επιπλέον, τις συναρτήσεις θα εκτελείται παράλληλα την ελαχιστοποίηση ο συνολικός χρόνος που απαιτείται για την επεξεργασία των αιτήσεων. Χρόνος εκτέλεσης για κάθε συνάρτηση προστίθεται προς τα επάνω, σε δευτερόλεπτα, και συναθροιστεί από την εφαρμογή που περιέχουν συνάρτηση. Με το κόστος καθοδήγηση μέσω με τον αριθμό των παρουσιών, το μέγεθος μνήμης και συνολικός χρόνος εκτέλεσης που μετράται σε Gigabyte δευτερόλεπτα. Αυτή είναι μια εξαιρετική επιλογή, εάν είναι περιοδικά τις ανάγκες σας υπολογισμού ή τις ώρες εργασίας την τάση να είναι πολύ σύντομο όπως σάς επιτρέπει να πληρώσετε μόνο για τους πόρους υπολογισμού όταν είναι στην πραγματικότητα χρησιμοποιείται.   

### <a name="memory-tier"></a>Επίπεδο μνήμης

Ανάλογα με το συνάρτηση πρέπει μπορείτε να επιλέξετε το μέγεθος της μνήμης που απαιτούνται για την εκτέλεση τους στην εφαρμογή συνάρτηση (κοντέινερ συναρτήσεων).
Οι επιλογές μεγέθους μνήμης διαφέρουν από **128 MB για 1536 MB**. Το μέγεθος της επιλεγμένης μνήμης αντιστοιχεί σε σύνολο εργασίας απαιτούνται από όλες τις συναρτήσεις που αποτελούν μέρος της εφαρμογής σας συνάρτηση. Εάν σας κώδικα απαιτεί περισσότερη μνήμη από το επιλεγμένο μέγεθος, την παρουσία εφαρμογή συνάρτηση θα τερματιστεί λόγω έλλειψης διαθέσιμης μνήμης.

### <a name="scaling"></a>Κλιμάκωση

Η πλατφόρμα συναρτήσεις Azure αξιολογεί την κίνηση ανάγκες σας, με βάση τα έχετε διαμορφώσει εναύσματα, να αποφασίσετε πότε να κλιμακωθεί προς τα επάνω ή προς τα κάτω. Το επίπεδο λεπτομερειών της κλίμακας είναι η εφαρμογή συνάρτηση. Κλιμάκωση προς τα επάνω σε αυτήν την περίπτωση σημαίνει ότι το προσθέσετε περισσότερες παρουσίες μιας εφαρμογής της συνάρτησης. Κόμβοι όπως μεταβαίνει κίνηση προς τα κάτω, συνάρτηση εφαρμογή παρουσίες είναι απενεργοποιημένη-κλίμακας προς τα κάτω τελικά σε μηδέν όταν εκτελούνται κανένα.  

### <a name="resource-consumption-and-billing"></a>Κατανάλωση πόρων και τις χρεώσεις

Στο η δυναμική λειτουργία εκχώρηση πόρων γίνεται με διαφορετικό τρόπο από το τυπικό πρόγραμμα εφαρμογής υπηρεσίας, επομένως το μοντέλο κατανάλωση επίσης είναι διαφορετικό, επιτρέποντάς για ένα μοντέλο "πληρωμή-ανά χρήση". Κατανάλωση θα αναφερθούν ανά εφαρμογή συνάρτησης, μόνο για το χρόνο όταν εκτελείται κώδικα.  
Υπολογίζεται με τον πολλαπλασιασμό το μέγεθος της μνήμης (σε GB) από το σύνολο του χρόνου εκτέλεσης (σε δευτερόλεπτα) για όλες τις συναρτήσεις εκτελούνται μέσα σε συγκεκριμένη συνάρτηση εφαρμογής. Η μονάδα κατανάλωση θα **GB-s (Gigabyte δευτερόλεπτα)**.
