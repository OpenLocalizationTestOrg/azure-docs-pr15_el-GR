# <a name="managing-alerts-with-microsoft-monitoring"></a>Διαχείριση ειδοποιήσεων με την παρακολούθηση της Microsoft 

Μια προειδοποίηση υποδεικνύει ορισμένες θέμα που απαιτεί την προσοχή από το διαχειριστή.  Υπάρχουν σημαντικές διαφορές μεταξύ του συστήματος κέντρο Operations Manager (SCOM) και τις αναλύσεις καταγραφής στην οικογένεια προγραμμάτων διαχείρισης λειτουργίες (OMS) όσον αφορά τον τρόπο δημιουργίας των ειδοποιήσεων, πώς γίνεται και ανάλυση και πώς θα ειδοποιείστε ότι εντοπίστηκε ένα κρίσιμο θέμα.

## <a name="alerts-in-operations-manager"></a>Ειδοποιήσεις στο Operations Manager
Ειδοποιήσεις στο SCOM δημιουργούνται από μεμονωμένους κανόνες ή των οθονών για να υποδείξετε ένα συγκεκριμένο ζήτημα.  Μια οθόνη μπορεί να δημιουργήσει μια ειδοποίηση όταν μεταβεί σε κατάσταση σφάλματος όταν ένας κανόνας μπορεί να δημιουργήσει μια ειδοποίηση για να υποδείξετε ότι ορισμένες κρίσιμο θέμα που δεν σχετίζονται απευθείας με την εύρυθμη λειτουργία της διαχειριζόμενο αντικείμενο.  Πακέτα διαχείρισης περιλαμβάνουν μια ποικιλία των ροών εργασιών που Δημιουργία ειδοποιήσεων για την εφαρμογή ή υπηρεσία, το οποίο διαχειρίζονται.  Μέρος της διαδικασίας της ρύθμισης των παραμέτρων ενός νέου πακέτου διαχείρισης ρύθμισης την για να βεβαιωθείτε ότι δεν έχετε λάβει υπερβολική ειδοποιήσεις για θέματα που δεν θεωρείτε κρίσιμη.

![Προβολή ειδοποίησης SCOM](media/operations-management-suite-monitoring-alerts/scom-alert-view.png)

SCOM παρέχει πλήρη ειδοποίησης διαχείρισης ειδοποιήσεων χρειάζεται μια κατάσταση που μπορεί να αλλάξει από τους διαχειριστές καθώς εργάζονται για να επιλύσετε το ζήτημα.  Όταν το ζήτημα έχει επιλυθεί, ο διαχειριστής ρυθμίζει την ειδοποίηση για να κλείσει, οπότε θα εμφανίζεται πλέον προβολές εμφάνιση ενεργού ειδοποιήσεις.  Ειδοποιήσεις που δημιουργούνται από οθόνες μπορούν να επιλυθούν αυτόματα όταν η οθόνη επιστρέφει σε καλή κατάσταση.

![Ειδοποίηση κατάστασης](media/operations-management-suite-monitoring-alerts/scom-alert-status.png)

## <a name="alerts-in-log-analytics"></a>Ειδοποιήσεις στο αρχείο καταγραφής ανάλυσης
Ειδοποίηση ανάλυση καταγραφής έχει δημιουργηθεί από ένα ερώτημα καταγραφής που εκτελείται αυτόματα σε τακτά χρονικά διαστήματα.  Μπορείτε να δημιουργήσετε έναν κανόνα ειδοποίησης από οποιοδήποτε ερώτημα καταγραφής.  Εάν το ερώτημα επιστρέφει αποτελέσματα που ικανοποιούν τα κριτήρια που καθορίζετε, στη συνέχεια, θα δημιουργηθεί μια ειδοποίηση.  Αυτό θα μπορούσε να ένα συγκεκριμένο ερώτημα που δημιουργεί μια ειδοποίηση, σε περίπτωση που εντοπιστεί ένα συγκεκριμένο συμβάν ή μπορείτε να χρησιμοποιήσετε ένα πιο γενικές ερώτημα που αναζητά οποιοδήποτε συμβάν σφάλματος που σχετίζονται με μια συγκεκριμένη εφαρμογή.

Ειδοποιήσεις ανάλυση καταγραφής εγγράφονται στο αποθετήριο OMS ως ένα συμβάν και μπορεί να ανακτηθεί με ένα ερώτημα αρχείου καταγραφής.  Δεν έχουν μια κατάσταση όπως SCOM συμβάντα, έτσι ώστε να μπορείτε να υποδείξετε όταν το ζήτημα έχει επιλυθεί.

![Ειδοποίηση OMS](media/operations-management-suite-monitoring-alerts/oms-alert.png)

Όταν SCOM χρησιμοποιείται ως προέλευση δεδομένων για το αρχείο καταγραφής ανάλυσης, ειδοποιήσεις SCOM εγγράφονται στο αποθετήριο OMS καθώς δημιουργούνται και τροποποίηση.  

![Ειδοποίηση SCOM](media/operations-management-suite-monitoring-alerts/scom-alert.png)

Η [λύση ειδοποίησης διαχείρισης](http://technet.microsoft.com/library/mt484092.aspx) παρέχει μια σύνοψη των ειδοποιήσεων ενεργό και διάφορες κοινά ερωτήματα για την ανάκτηση διαφορετικά σύνολα ειδοποιήσεις.  Αυτό σας παρέχει πιο αποτελεσματική ανάλυση των τις ειδοποιήσεις σας από μια αναφορά σε SCOM.  Μπορείτε να κάνετε Διερεύνηση σε από τις συνόψεις με λεπτομερή δεδομένα και να δημιουργήσετε ad hoc ερωτημάτων για να ανακτήσετε διαφορετικά σύνολα των ειδοποιήσεων.

![Λύση ειδοποίησης διαχείρισης](media/operations-management-suite-monitoring-alerts/alert-management.png)

## <a name="notifications"></a>Ειδοποιήσεις
Ειδοποιήσεις στο SCOM αποστολή που έναν αλληλογραφίας ή κείμενο σε απάντηση σε ειδοποιήσεις που συμφωνούν με συγκεκριμένο κριτήριο.  Μπορείτε να δημιουργήσετε συνδρομές διαφορετικό ειδοποίησης που έχουν διαφορετικά άτομα ειδοποιηθούν ανάλογα με κριτήρια όπως το αντικείμενο που παρακολουθούνται, της σοβαρότητας των την ειδοποίηση, το είδος του προβλήματος που εντοπίστηκε ή την ώρα της ημέρας.

Μερικά συνδρομές μπορεί να χρησιμοποιηθεί για την υλοποίηση της στρατηγικής πλήρους κοινοποίησης για μεγάλο αριθμό πακέτα διαχείρισης.

![Ειδοποιήσεις SCOM](media/operations-management-suite-monitoring-alerts/alerts-overview-scom.png)

Ανάλυση καταγραφής μπορεί να σας ειδοποιεί μέσω ηλεκτρονικού ταχυδρομείου που έχει δημιουργηθεί μια ειδοποίηση, ορίζοντας μια ενέργεια ειδοποίησης ηλεκτρονικού ταχυδρομείου σε κάθε [ειδοποίησης κανόνα](http://technet.microsoft.com/library/mt614775.aspx).  Να μην έχει τη δυνατότητα των SCOM την εγγραφή σε πολλαπλές προειδοποιήσεις με έναν κανόνα.  Πρέπει επίσης να δημιουργήσετε το δικό σας ειδοποίησης κανόνες καθώς OMS δεν παρέχει οποιαδήποτε προ-διαμορφωμένες.

![Ειδοποιήσεις ανάλυσης αρχείου καταγραφής](media/operations-management-suite-monitoring-alerts/alerts-overview-oms.png)

Πλήρως δεν μπορείτε να διαχειριστείτε τις ειδοποιήσεις SCOM στο αρχείο καταγραφής αναλυτικών στοιχείων μέσω δεδομένου ότι μπορείτε να τους τροποποιήσετε μόνο στην κονσόλα λειτουργίες.  Αρχείο καταγραφής ανάλυσης είναι χρήσιμο ως μέρος μιας ειδοποίησης διαχείρισης μέσω διαδικασίας για παροχή εργαλεία ανάλυσης που SCOM από μόνο του δεν διαθέτει.

## <a name="alert-remediation"></a>Ειδοποίηση αποκατάσταση εύρυθμης λειτουργίας
[Αποκατάσταση εύρυθμης λειτουργίας](http://technet.microsoft.com/library/mt614775.aspx) αναφέρεται σε μιας προσπάθειας για αυτόματη διόρθωση του προβλήματος που προσδιορίζεται από μια ειδοποίηση.
  
SCOM σάς επιτρέπει να εκτελέσετε διαγνωστικά και ανακτήσεις απόκριση σε μια οθόνη εισαγωγή κατάσταση εύρυθμης λειτουργίας.  Αυτό συμβαίνει ταυτόχρονη στην οθόνη δημιουργία την ειδοποίηση.  Εργαλεία διαγνωστικών και ανακτήσεις υλοποιούνται συνήθως ως μια δέσμη ενεργειών που εκτελείται σε τον παράγοντα.  Μια diagnostic προσπαθεί να συγκεντρώσετε περισσότερες πληροφορίες σχετικά με το Εντοπίστηκε πρόβλημα ενώ αποκατάστασης προσπαθεί να διορθώσετε το πρόβλημα.

Ανάλυση καταγραφής σάς επιτρέπει να Έναρξη μια [runbook αυτοματισμού Azure](https://azure.microsoft.com/documentation/services/automation/) ή κλήσης webhook μια απόκριση σε μια ειδοποίηση καταγραφής ανάλυσης.  Runbooks μπορούν να περιέχουν περίπλοκη λογική υλοποιηθεί σε PowerShell.  Η δέσμη ενεργειών εκτελείται στο Azure και πρόσβαση σε οποιαδήποτε Azure πόρους ή εξωτερικοί πόροι που είναι διαθέσιμα από το cloud.  Azure αυτοματισμού έχουν τη δυνατότητα να εκτελέσει runbooks σε ένα διακομιστή του τοπικού κέντρου δεδομένων, αλλά αυτή η δυνατότητα δεν είναι διαθέσιμη αυτήν τη στιγμή κατά την εκκίνηση του runbook απόκριση σε ειδοποιήσεις καταγραφής ανάλυσης.

Δύο ανακτήσεις στο SCOM και runbooks στο OMS μπορούν να περιέχουν δεσμών ενεργειών του PowerShell, αλλά ανακτήσεις είναι δυσκολότερο να δημιουργήσετε και να διαχειριστείτε επειδή πρέπει να περιέχεται μέσα σε ένα πακέτο διαχείρισης.  Runbooks αποθηκεύονται στην αυτοματοποίηση Azure που παρέχει δυνατότητες για σύνταξη από κοινού, δοκιμή και τη Διαχείριση runbooks.

Εάν χρησιμοποιείτε το SCOM ως προέλευση δεδομένων για το αρχείο καταγραφής ανάλυσης, μπορείτε να δημιουργήσετε μια ειδοποίηση ανάλυση καταγραφής χρησιμοποιώντας ένα ερώτημα καταγραφής για την ανάκτηση SCOM ειδοποιήσεων που είναι αποθηκευμένα στο αποθετήριο OMS.  Αυτό θα σας επιτρέψει να εκτελέσετε μια runbook αυτοματισμού Azure απόκριση σε μια ειδοποίηση SCOM.  Φυσικά, επειδή το runbook θα εκτελείται σε Azure, αυτό δεν θα βιώσιμων στρατηγικής για ανακτήσεις ζητημάτων εσωτερικής εγκατάστασης.
