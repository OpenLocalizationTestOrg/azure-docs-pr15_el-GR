## <a name="scenario"></a>Σενάριο

Το ακόλουθο σενάριο μεταβαίνει στη δημιουργία ενός προσαρμοσμένου εύρυθμης λειτουργίας δοκιμή του σε μια υπάρχουσα πύλη εφαρμογής.
Το σενάριο προϋποθέτει ότι έχετε ήδη ακολουθήσατε τα βήματα για να [δημιουργήσετε μια πύλη εφαρμογής](application-gateway-create-gateway-portal.md).

## <a name="createprobe"></a>Δημιουργήστε τη διερεύνηση

Καθετήρες έχουν ρυθμιστεί οι παράμετροι σε μια διαδικασία δύο βημάτων μέσω της πύλης. Το πρώτο βήμα είναι να δημιουργήσετε τη δοκιμή του, στη συνέχεια μπορείτε να προσθέσετε τη δοκιμή του τις ρυθμίσεις http παρασκηνίου της πύλης εφαρμογής.

### <a name="step-1"></a>Βήμα 1

Μεταβείτε στο http://portal.azure.com και επιλέξτε μια υπάρχουσα πύλη εφαρμογής.

![Επισκόπηση της εφαρμογής πύλης][1]

### <a name="step-2"></a>Βήμα 2

Κάντε κλικ στην επιλογή **Probes** και κάντε κλικ στο κουμπί **Προσθήκη** για να προσθέσετε μια νέα δοκιμή του.

![Προσθήκη δοκιμή του blade με πληροφορίες συμπληρωμένο][2]

### <a name="step-3"></a>Βήμα 3

Συμπληρώστε τις απαιτούμενες πληροφορίες για τη δοκιμή του και όταν τελειώσετε, κάντε κλικ στο κουμπί **OK**.

- **Όνομα** - αυτό είναι ένα φιλικό όνομα για τη δοκιμή του που είναι δυνατή η πρόσβαση στην πύλη.
- **Κεντρικός υπολογιστής** - αυτό είναι το όνομα κεντρικού υπολογιστή που χρησιμοποιείται για τη δοκιμή του. Ισχύει μόνο όταν πολλών τοποθεσιών έχει ρυθμιστεί στην πύλη εφαρμογής, διαφορετικά χρησιμοποιείτε "127.0.0.1". Αυτό είναι διαφορετικό από το όνομα κεντρικού υπολογιστή Εικονική.
- **Διαδρομή** - το υπόλοιπο της πλήρους διεύθυνσης url για την προσαρμοσμένη δοκιμή του. Μια έγκυρη διαδρομή ξεκινά με "/"
- **Χρονικό διάστημα (δευτ.)** - πόσο συχνά εκτελείται η δοκιμή του για να ελέγξετε για εύρυθμης λειτουργίας. Δεν συνιστάται για να ορίσετε την κατώτερη από 30 δευτερόλεπτα.
- **Χρονικό όριο (δευτ.)** - το χρονικό διάστημα που χρειάζεται να περιμένει τη δοκιμή του πριν λήξει το χρονικό όριο. Το χρονικό διάστημα πρέπει να είναι αρκετά υψηλή ώστε μια κλήση http μπορούν να γίνουν για να βεβαιωθείτε ότι η σελίδα εύρυθμη λειτουργία παρασκηνίου είναι διαθέσιμη.
- **Κατεστραμμένη όριο** - αριθμός των αποτυχημένων προσπαθειών πρέπει να θεωρούνται κατεστραμμένες. Μια οριακή τιμή 0 σημαίνει ότι, εάν μια αποτυγχάνει ελέγχου εύρυθμης λειτουργίας του παρασκηνίου θα προσδιοριστεί κατεστραμμένες αμέσως.

> [AZURE.IMPORTANT] το όνομα κεντρικού υπολογιστή δεν είναι το όνομα του διακομιστή. Αυτό είναι το όνομα του κεντρικού υπολογιστή εικονικού εκτελείται στο διακομιστή εφαρμογών. Η δοκιμή του αποστέλλεται http://(host name):(port from httpsetting)/urlPath

![ρυθμίσεις παραμέτρων δοκιμή του][3]

## <a name="add-probe-to-the-gateway"></a>Προσθήκη δοκιμή του στην πύλη

Τώρα που έχει δημιουργηθεί η δοκιμή του, είναι ώρα για να την προσθέσετε στην πύλη. Δοκιμή του ρυθμίσεις έχουν οριστεί σχετικά με τις ρυθμίσεις http παρασκηνίου της εφαρμογής πύλης.

### <a name="step-1"></a>Βήμα 1

Κάντε κλικ στις **Ρυθμίσεις HTTP** της πύλης εφαρμογής και, στη συνέχεια, κάντε κλικ στην επιλογή τις τρέχουσες παρασκηνίου http ρυθμίσεις στο παράθυρο για να εμφανίσετε το blade ρύθμισης παραμέτρων.

![παράθυρο ρυθμίσεων HTTPS][4]

### <a name="step-2"></a>Βήμα 2

Από το blade ρυθμίσεις **appGatewayBackEndHttp** , κάντε κλικ στην επιλογή **Δοκιμή του προσαρμοσμένου χρήση** και επιλέξτε τη δοκιμή του που δημιουργήσατε στην ενότητα [Δημιουργία τη δοκιμή του](#createprobe) .
Όταν τελειώσετε, κάντε κλικ στο **κουμπί OK** και να εφαρμοστούν οι ρυθμίσεις.

![appgatewaybackend ρυθμίσεις blade][5]

Η δοκιμή του προεπιλεγμένου ελέγχει την προεπιλεγμένη πρόσβαση στην εφαρμογή web. Τώρα που έχει δημιουργηθεί ένα προσαρμοσμένο Διερεύνηση, η πύλη εφαρμογής χρησιμοποιεί προσαρμοσμένη διαδρομή που ορίζεται για την παρακολούθηση της εύρυθμης λειτουργίας για την επιλεγμένη παρασκηνίου. Με βάση τα κριτήρια που ορίστηκε, η πύλη εφαρμογής ελέγχει το αρχείο που καθορίζεται στο τη διερεύνηση. Εάν η κλήση κεντρικού υπολογιστή: θύρα / διαδρομή δεν επιστρέφει μια απόκριση κατάστασης Http 200 OK, ο διακομιστής λαμβάνεται από περιστροφής, μετά την κατεστραμμένη κατώτερο όριο. Η διερεύνηση συνεχίζεται στην κατεστραμμένες παρουσία για να καθορίσετε πότε θα μετατραπεί σε καλή κατάσταση ξανά. Μόλις προστεθεί ξανά την παρουσία στο διακομιστή σε καλή κατάσταση συγκέντρωσης κίνηση αρχίζει ροή σε αυτήν ξανά και Διερεύνηση για να την παρουσία συνεχίζει στο καθορισμένο χρονικό διάστημα χρήστη κανονικά.
