# <a name="release-notes-for-hadoop-components-on-azure-hdinsight"></a>Σημειώσεις έκδοσης για στοιχεία Hadoop Azure HDInsight

## <a name="notes-for-10262016-release-of-r-server-on-hdinsight"></a>Σημειώσεις για την έκδοση του διακομιστή R στο HDInsight 26/10/2016

- Έχει βελτιστοποιηθεί R διακομιστή στο προμήθεια σύμπλεγμα HDInsight.
- R διακομιστή στο HDInsight είναι πλέον διαθέσιμη ως κανονική HDInsight "R διακομιστή" συμπλέγματος τύπος και δεν είναι πλέον εγκατασταθεί ως ξεχωριστή εφαρμογή HDInsight. Τον κόμβο άκρου και δυαδικά δεδομένα διακομιστή R τώρα έχουν παρασχεθεί ως μέρος της ανάπτυξης συμπλέγματος διακομιστή R. Αυτό βελτιώνει την ταχύτητα και την αξιοπιστία των προμήθειας. Μοντέλο τιμολόγησης για διακομιστή R είναι ενημερωθεί αντίστοιχα.
- Τιμή τύπου σύμπλεγμα διακομιστή R τώρα βασίζεται σε τιμή Standard επίπεδο συν τιμή πρόσθετης R διακομιστή. Premium επίπεδο τώρα θα προορίζεται για κορυφαίες δυνατότητες που είναι διαθέσιμες σε σύμπλεγμα διαφορετικούς τύπους και δεν θα χρησιμοποιηθεί για τον τύπο σύμπλεγμα διακομιστή R. Αυτή η αλλαγή δεν επηρεάζει την αποτελεσματική τις τιμές του διακομιστή R, αλλάζει μόνο με ποιον τρόπο θα παρουσιάζονται τις χρεώσεις στο τιμολόγιο. Όλα υπάρχοντα συμπλεγμάτων R Server θα συνεχίσουν να λειτουργούν και πρότυπα ARM θα συνεχίσει να λειτουργεί μέχρι να υπάρξει με ειδοποίηση. **Συνιστάται αν και να ενημερώσετε τις δέσμες ενεργειών αναπτύξεις για να χρησιμοποιήσετε το νέο πρότυπο ARM.**

Αυτά τα θέματα συμβατότητας επιλύονται στις πιο πρόσφατες εκδόσεις των HDInsight SDK και Azure PowerShell. Σας συνιστούμε να ενημερώνετε το HDInsight SDK σε έκδοση 1.3.1.6 ή νεότερη έκδοση και Azure εργαλεία PowerShell έκδοση 0.8.8 ή νεότερη έκδοση. ### <a name="hive-queries-can-use-tez-in-hdinsight-31"></a>Ομάδα ερωτήματα, να χρησιμοποιήσετε Tez στο HDInsight 3.1

Ομάδα 0,13 είναι διαθέσιμες στο HDInsight 3.1 και μπορεί να εκτελεί ερωτήματα με Tez, τα οποία μπορούν να χρησιμοποιηθούν για βελτιώσεις απόδοσης ουσιαστικά.
Tez δεν είναι ενεργοποίηση από προεπιλογή για ερωτήματα ομάδας. Για να χρησιμοποιήσετε, που πρέπει να επιλέξετε. Μπορείτε να ενεργοποιήσετε Tez, εκτελώντας το ακόλουθο κώδικα:

        set hive.execution.engine=tez;
        select sc_status, count(*), histogram_numeric(sc_bytes,5) from website_logs_orc_local group by sc_status;

Hortonworks έχει δημοσιεύσει μια λεπτομερή ανάλυση του Hive βελτιώσεις επιδόσεων ερωτήματος με Tez όπως παραδίδεται σε τυπική σημεία αναφοράς. Για περισσότερες λεπτομέρειες σχετικά με τη χρήση της ομάδας με Tez, ανατρέξτε στο θέμα [Hive στην Tez](https://cwiki.apache.org/confluence/display/Hive/Hive+on+Tez).

###<a name="dos--donts-between-cluster-versions"></a>Οδηγίες και μη του μεταξύ των εκδόσεων συμπλέγματος

**Χρησιμοποιείται με ένα σύμπλεγμα HDInsight 3.1 metastores Oozie είναι συμβατά με παλαιότερες εκδόσεις δεν με συμπλεγμάτων HDInsight 2.1, και δεν μπορούν να χρησιμοποιηθούν με αυτήν την προηγούμενη έκδοση**.

Δεν είναι δυνατό να χρησιμοποιηθεί ξανά αναπτυχθεί με ένα σύμπλεγμα HDInsight 3.1 μια προσαρμοσμένη βάση δεδομένων metastore Oozie με ένα σύμπλεγμα HDInsight 2.1. Αυτό συμβαίνει ακόμα και αν το metastore δημιουργήθηκε με ένα σύμπλεγμα HDInsight 2.1. Αυτό το σενάριο δεν υποστηρίζεται, επειδή το σχήμα metastore λαμβάνει αναβάθμιση όταν χρησιμοποιείται με ένα σύμπλεγμα HDInsight 3.1, ώστε να μην είναι πλέον συμβατά με το metastore που απαιτούνται από το HDInsight 2.1 συμπλεγμάτων. Οποιαδήποτε προσπάθεια να επαναχρησιμοποιήσετε μια metastore Oozie που έχει χρησιμοποιηθεί με ένα σύμπλεγμα HDInsight 3.1 θα απόδοση του συμπλέγματος HDInsight 2.1 άχρηστο.

**Δεν είναι δυνατή η κοινή χρήση Oozie metastores μεταξύ συμπλεγμάτων.**

Oozie metastores επισυνάπτονται σε συγκεκριμένα συμπλεγμάτων και δεν είναι δυνατό να έθεσε μεταξύ συμπλεγμάτων.

###<a name="breaking-changes"></a>Διακοπή αλλαγές

**Πρόθεμα σύνταξη**: μόνο το "wasbs: / /" σύνταξη υποστηρίζεται στο HDInsight 3.1 και 3.0 συμπλεγμάτων. Το παλαιότερο "asv: / /" σύνταξη υποστηρίζεται στο HDInsight 2.1 και 1,6 συμπλεγμάτων, αλλά δεν υποστηρίζεται σε HDInsight 3.1 ή 3.0 συμπλεγμάτων. Αυτό σημαίνει ότι υπάρχουν εργασίες που υποβάλλεται σε ένα 3.1 HDInsight ή 3.0 σύμπλεγμα που χρησιμοποιούν ρητά την "asv: / /" σύνταξη θα αποτύχει. Το "wasbs: / /" σύνταξη θα πρέπει να χρησιμοποιείται αντί για αυτό. Επίσης, οι εργασίες που υποβάλλεται σε οποιαδήποτε 3.1 HDInsight ή 3.0 συμπλεγμάτων που έχουν δημιουργηθεί με μια υπάρχουσα metastore που περιέχει ρητές αναφορές σε πόρους χρησιμοποιώντας την "asv: / /" σύνταξη θα αποτύχει. Αυτά τα metastores πρέπει να δημιουργηθούν εκ νέου χρήση του "wasbs: / /" σύνταξη με διεύθυνση πόρους.
