# <a name="known-issues-for-apache-spark-cluster-on-hdinsight-linux"></a>Γνωστά θέματα για τους Apache σύμπλεγμα σε HDInsight Linux

Αυτό το έγγραφο παρακολουθεί των όλα τα γνωστά θέματα για την προεπισκόπηση δημόσια HDInsight τους.  

##<a name="livy-leaks-interactive-session"></a>Λίβιος είναι αλληλεπιδραστική περίοδο λειτουργίας
 
Κατά την επανεκκίνηση Λίβιος με μια διαδραστική περίοδο λειτουργίας (από Ambari ή λόγω επανεκκίνηση του υπολογιστή εικονικές headnode 0) ακόμη εν ζωή, θα είναι διαρροής μια περίοδο λειτουργίας αλληλεπιδραστικών εργασία. Αυτόν το λόγο, νέες εργασίες μπορεί να έχει κολλήσει σε κατάσταση αποδεκτοί και δεν μπορεί να ξεκινήσει.

**Μετριασμού:**

Χρησιμοποιήστε την ακόλουθη διαδικασία για να επιλύσετε το πρόβλημα:

1. SSH σε headnode. 
2. Εκτελέστε την παρακάτω εντολή για να βρείτε τα αναγνωριστικά εφαρμογής των εργασιών αλληλεπιδραστικών αποτελέσματα μέσω Λίβιος. 

        yarn application –list

    Η εργασία προεπιλεγμένη ονόματα θα Λίβιος αν οι εργασίες έχουν ξεκινήσει με μια περίοδο λειτουργίας Λίβιος αλληλεπιδραστικών χωρίς ρητή ονόματα που καθορίζονται, την περίοδο λειτουργίας για Λίβιος ξεκίνησε από Jupyter Σημειωματάριο, το όνομα της εργασίας θα ξεκινήσει με remotesparkmagics_ *. 

3. Εκτελέστε την παρακάτω εντολή για να τερματισμού αυτές οι εργασίες. 

        yarn application –kill <Application ID>

Νέες εργασίες θα ξεκινήσει η εκτέλεση. 

##<a name="spark-history-server-not-started"></a>Διακομιστής ιστορικού τους δεν ξεκίνησε 

Τους ιστορικού Server δεν έχει ξεκινήσει αυτόματα, αφού δημιουργηθεί ένα σύμπλεγμα.  

**Μετριασμού:** 

Ξεκινήστε με μη αυτόματο τρόπο το ιστορικό διακομιστή από Ambari.

## <a name="permission-issue-in-spark-log-directory"></a>Το ζήτημα δικαιωμάτων στον κατάλογο καταγραφής τους 

Όταν hdiuser υποβάλλει μια εργασία με spark-submit, υπάρχει μια java.io.FileNotFoundException σφάλμα: δεν εγγράφεται /var/log/spark/sparkdriver_hdiuser.log (δεν έχετε δικαίωμα πρόσβασης) και το αρχείο καταγραφής του προγράμματος οδήγησης. 

**Μετριασμού:**
 
1. Προσθήκη hdiuser στην ομάδα Hadoop. 
2. Παρέχει 777 δικαιώματα /var/log/spark μετά τη δημιουργία του συμπλέγματος. 
3. Ενημερώστε τη θέση του αρχείου καταγραφής τους χρησιμοποιώντας Ambari για να είναι ένας κατάλογος με 777 δικαιώματα.  
4. Εκτέλεση τους-υποβολής ως sudo.  

## <a name="issues-related-to-jupyter-notebooks"></a>Ζητήματα που σχετίζονται με σημειωματάρια Jupyter

Ακολουθούν ορισμένα γνωστά θέματα που σχετίζονται με Jupyter σημειωματάρια.


### <a name="notebooks-with-non-ascii-characters-in-filenames"></a>Τα σημειωματάρια με μη ASCII χαρακτήρες σε ονόματα αρχείων

Jupyter τα σημειωματάρια που μπορεί να χρησιμοποιηθεί σε συμπλεγμάτων HDInsight τους δεν θα πρέπει να έχετε μη ASCII χαρακτήρες σε ονόματα αρχείων. Εάν προσπαθείτε να αποστείλετε ένα αρχείο μέσω του περιβάλλοντος εργασίας Χρήστη Jupyter που έχει ένα όνομα αρχείου δεν είναι ASCII, θα αποτύχει σιωπηρά (δηλαδή, Jupyter δεν σας επιτρέπουν να αποστείλετε το αρχείο, αλλά αυτό δεν θα εμφανίσουν είτε ένα ορατό σφάλμα). 

### <a name="error-while-loading-notebooks-of-larger-sizes"></a>Σφάλμα κατά τη φόρτωση σημειωματάρια μεγαλύτερα μεγέθη

Ενδέχεται να εμφανιστεί ένα σφάλμα **`Error loading notebook`** πότε να φορτώσετε τα σημειωματάρια που είναι μεγαλύτερα σε μέγεθος.  

**Μετριασμού:**

Εάν λάβετε αυτό το σφάλμα, αυτό δεν σημαίνει των δεδομένων σας έχει καταστραφεί ή χαθεί.  Τα σημειωματάρια είναι ακόμη σε δίσκο σε `/var/lib/jupyter`, και μπορείτε να SSH σε σύμπλεγμα για να αποκτήσετε πρόσβαση σε αυτά. Μπορείτε να αντιγράψετε τα σημειωματάριά σας από το σύμπλεγμά σας στον τοπικό υπολογιστή (χρησιμοποιώντας SCP ή WinSCP) ως αντίγραφο ασφαλείας για να αποτρέψετε την απώλεια των σημαντικών δεδομένων στο Σημειωματάριο. Μπορείτε, στη συνέχεια, να SSH σωλήνα σε σας headnode στη θύρα 8001 για να αποκτήσετε πρόσβαση Jupyter χωρίς να μεταβείτε από την πύλη.  Από εκεί, μπορείτε να καταργήσετε το αποτέλεσμα του σημειωματαρίου σας και αποθηκεύστε ξανά για να ελαχιστοποιήσετε το μέγεθος του σημειωματαρίου.

Για να αποφύγετε αυτό το σφάλμα συμβαίνει στο μέλλον, πρέπει να ακολουθήσετε ορισμένες βέλτιστες πρακτικές:

* Είναι σημαντικό να διατηρήσετε μικρό το μέγεθος του σημειωματαρίου. Κανένα αποτέλεσμα από τις εργασίες τους που αποστέλλεται στη Jupyter παραμένει στο Σημειωματάριο.  Πρόκειται για βέλτιστη πρακτική με Jupyter σε γενικές γραμμές για να αποφύγετε `.collect()` σε μεγάλο RDD του ή dataframes; αντί για αυτό, εάν θέλετε να Ρίξτε μια ματιά σε ένα RDD περιεχόμενα, εξετάστε το ενδεχόμενο χρήσης του `.take()` ή `.sample()` έτσι ώστε το αποτέλεσμα δεν πολύ μεγάλο.
* Επίσης, όταν αποθηκεύετε ένα σημειωματάριο, καταργήστε την επιλογή όλες τις εξόδου κελιών για να μειώσετε το μέγεθος.

### <a name="notebook-initial-startup-takes-longer-than-expected"></a>Αρχική εκκίνηση Σημειωματάριο διαρκεί περισσότερο από το αναμενόμενο 

Η πρώτη πρόταση κώδικα στο Σημειωματάριο Jupyter χρησιμοποιώντας τους μαγικός μπορεί να διαρκέσει περισσότερο από ένα λεπτό.  

**Εξήγηση:**
 
Αυτό συμβαίνει επειδή όταν εκτελείται το πρώτο κελί κώδικα. Στο παρασκήνιο ενεργοποιείται η δυνατότητα ρύθμισης παραμέτρων περιόδου λειτουργίας και τους, SQL και ορίζονται Hive περιβάλλοντα. Αφού έχουν οριστεί αυτά τα περιβάλλοντα, στον οποίο εκτελείται η πρώτη πρόταση και αυτό σας δίνει την εντύπωση που τη δήλωση διαρκεί πολύ χρόνο για να ολοκληρώσετε.

### <a name="jupyter-notebook-timeout-in-creating-the-session"></a>Το χρονικό όριο Σημειωματάριο Jupyter στη δημιουργία της περιόδου λειτουργίας

Όταν τους σύμπλεγμα είναι εκτός των πόρων, την πυρήνων τους και Pyspark στο Σημειωματάριο Jupyter θα χρονικού ορίου που προσπαθείτε να δημιουργήσετε την περίοδο λειτουργίας. 

**Μετριασμούς:** 

1. Απελευθέρωση ορισμένους πόρους στο σύμπλεγμά σας τους από:

    - Διακοπή άλλων σημειωματαρίων τους μεταβαίνοντας στο μενού Κλείσιμο και διακόπτει ή κάνοντας κλικ τερματισμού στην Εξερεύνηση του σημειωματαρίου.
    - Διακοπή από ΝΉΜΑΤΑ από τις άλλες εφαρμογές τους.

2. Επανεκκινήστε το Σημειωματάριο που προσπαθήσατε να ξεκινήσετε. Πρέπει να είναι διαθέσιμη για να δημιουργήσετε μια περίοδο λειτουργίας τώρα αρκετούς πόρους.
