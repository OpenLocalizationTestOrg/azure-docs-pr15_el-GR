# <a name="datasets-in-azure-data-factory"></a>Σύνολα δεδομένων στην προέλευση δεδομένων Azure
Σε αυτό το άρθρο περιγράφει συνόλων δεδομένων στην προέλευση δεδομένων Azure και περιλαμβάνει παραδείγματα όπως offset, anchorDateTime και μετατόπιση/στυλ βάσεις δεδομένων.

Όταν δημιουργείτε ένα σύνολο δεδομένων, δημιουργείτε ένα δείκτη προς τα δεδομένα που θέλετε να επεξεργαστείτε. Επεξεργασίας δεδομένων (εισόδου/εξόδου) σε μια δραστηριότητα και μια δραστηριότητα περιέχονται σε μια διαδικασία. Ένα σύνολο δεδομένων εισαγωγής αντιπροσωπεύει την είσοδο για μια δραστηριότητα στη διοχέτευση και ένα σύνολο δεδομένων εξόδου αντιπροσωπεύει την έξοδο για τη δραστηριότητα.

Σύνολα δεδομένων τον εντοπισμό δεδομένων χώρους αποθήκευσης διαφορετικά δεδομένα, όπως πίνακες, αρχεία, φακέλους και έγγραφα. Αφού δημιουργήσετε ένα σύνολο δεδομένων, μπορείτε να το χρησιμοποιήσετε με δραστηριότητες σε μια διαδικασία. Για παράδειγμα, ένα σύνολο δεδομένων μπορεί να είναι ένα σύνολο δεδομένων εισόδου/εξόδου μιας δραστηριότητας αντίγραφο ή μια δραστηριότητα HDInsightHive. Πύλη του Azure παρέχει μια οπτική διάταξη όλων των σας αγωγούς και δεδομένων εισροές και εκροές. Με μια ματιά, βλέπετε όλες τις σχέσεις και τις εξαρτήσεις σας αγωγών όλες τις πηγές σας, ώστε να γνωρίζετε πάντα όπου προέρχεται δεδομένων και πού πρόκειται.

Στο Azure εργοστασίου δεδομένων, μπορείτε να λάβετε δεδομένα από ένα σύνολο δεδομένων, χρησιμοποιώντας δραστηριότητας αντίγραφο σε μια διαδικασία.

> [AZURE.NOTE] Εάν είστε νέος χρήστης του Azure εργοστασίου δεδομένων, ανατρέξτε στο θέμα [Εισαγωγή στις εργοστασιακές Azure δεδομένων](data-factory-introduction.md) για μια επισκόπηση των υπηρεσιών Azure εργοστασίου δεδομένων. Ανατρέξτε στο θέμα [Δημιουργία του πρώτου εργοστασίου δεδομένων](data-factory-build-your-first-pipeline.md) για ένα πρόγραμμα εκμάθησης για να δημιουργήσετε το πρώτο εργοστασίου δεδομένων. Αυτά τα δύο άρθρα παρέχουν φόντου πληροφορίες που χρειάζεστε για να κατανοήσετε καλύτερα σε αυτό το άρθρο. 

## <a name="define-datasets"></a>Ορισμός συνόλων δεδομένων
Ένα σύνολο δεδομένων στην προέλευση δεδομένων Azure ορίζεται ως εξής: 


    {
        "name": "<name of dataset>",
        "properties": {
            "type": "<type of dataset: AzureBlob, AzureSql etc...>",
            "external": <boolean flag to indicate external data. only for input datasets>,
            "linkedServiceName": "<Name of the linked service that refers to a data store.>",
            "structure": [
                {
                    "name": "<Name of the column>",
                    "type": "<Name of the type>"
                }
            ],
            "typeProperties": {
                "<type specific property>": "<value>",
                "<type specific property 2>": "<value 2>",
            },
            "availability": {
                "frequency": "<Specifies the time unit for data slice production. Supported frequency: Minute, Hour, Day, Week, Month>",
                "interval": "<Specifies the interval within the defined frequency. For example, frequency set to 'Hour' and interval set to 1 indicates that new data slices should be produced hourly>"
            },
           "policy": 
            {      
            }
        }
    }

Ο παρακάτω πίνακας περιγράφει τις ιδιότητες σε τα παραπάνω JSON:   

| Ιδιότητα | Περιγραφή | Απαιτείται | Προεπιλεγμένη |
| -------- | ----------- | -------- | ------- |
| Όνομα | Όνομα του συνόλου δεδομένων. Ανατρέξτε στο θέμα [Factory δεδομένων Azure - ονομασία κανόνες](data-factory-naming-rules.md) για τους κανόνες ονοματοθεσίας. | Ναι | NA |
| Τύπος | Τύπος του συνόλου δεδομένων. Καθορίστε έναν από τους τύπους που υποστηρίζονται από το Azure εργοστασίου δεδομένων (για παράδειγμα: AzureBlob, AzureSqlTable). <br/><br/>Για λεπτομέρειες, ανατρέξτε στο θέμα [Τύπου συνόλου δεδομένων](#Type) . | Ναι | NA |
| δομή | Σχήμα του συνόλου δεδομένων<br/><br/>Για λεπτομέρειες, ανατρέξτε στο θέμα [Δομή Dataset](#Structure) ενότητας. | Όχι. | NA |
| typeProperties | Ιδιότητες που αντιστοιχεί στον επιλεγμένο τύπο. Ανατρέξτε στην ενότητα [Τύπος σύνολο δεδομένων](#Type) για λεπτομέρειες σχετικά με το υποστηριζόμενοι τύποι και τις ιδιότητές τους. | Ναι | NA |
| εξωτερική | Δυαδική σημαία για να καθορίσετε εάν ένα σύνολο δεδομένων ρητά παράγεται από μια διοχέτευση εργοστασίου δεδομένων ή όχι.  | Όχι | FALSE | 
| διαθεσιμότητα | Καθορίζει το παράθυρο επεξεργασίας ή το μοντέλο slicing για την παραγωγή συνόλου δεδομένων. <br/><br/>Για λεπτομέρειες, ανατρέξτε στην ενότητα [Διαθεσιμότητα Dataset](#Availability) ενότητας. <br/><br/>Για λεπτομέρειες σχετικά με το σύνολο δεδομένων σε φέτες μοντέλο, ανατρέξτε στο θέμα [Προγραμματισμός και εκτέλεσης](data-factory-scheduling-and-execution.md) άρθρο. | Ναι | NA
| πολιτική | Καθορίζει τα κριτήρια ή τη συνθήκη που πρέπει να ικανοποιεί τις φέτες του συνόλου δεδομένων. <br/><br/>Για λεπτομέρειες, ανατρέξτε στο θέμα [Πολιτική Dataset](#Policy) ενότητας. | Όχι | NA |

## <a name="dataset-example"></a>Παράδειγμα συνόλου δεδομένων
Στο παρακάτω παράδειγμα, του συνόλου δεδομένων αντιπροσωπεύει έναν πίνακα που ονομάζεται **Πίνακας** σε βάση **βάση δεδομένων Azure SQL**. 

    {
        "name": "DatasetSample",
        "properties": {
            "type": "AzureSqlTable",
            "linkedServiceName": "AzureSqlLinkedService",
            "typeProperties": 
            {
                "tableName": "MyTable"
            },
            "availability": 
            {
                "frequency": "Day",
                "interval": 1
            }
        }
    }

Λάβετε υπόψη τα εξής σημεία: 

- Τύπος έχει οριστεί σε AzureSqlTable.
- ιδιότητα τύπος όνομα πίνακα (συγκεκριμένους τύπους AzureSqlTable) έχει οριστεί σε πίνακας.
- linkedServiceName αναφέρεται σε ένα συνδεδεμένο της υπηρεσίας τύπου AzureSqlDatabase. Δείτε τον ορισμό της παρακάτω συνδεδεμένων υπηρεσίας. 
- διαθεσιμότητα συχνότητα έχει οριστεί σε ημέρα και διάστημα έχει οριστεί σε 1, γεγονός που σημαίνει ότι η φέτα παράγεται καθημερινά.  

AzureSqlLinkedService ορίζεται ως εξής:

    {
        "name": "AzureSqlLinkedService",
        "properties": {
            "type": "AzureSqlDatabase",
            "description": "",
            "typeProperties": {
                "connectionString": "Data Source=tcp:<servername>.database.windows.net,1433;Initial Catalog=<databasename>;User ID=<username>@<servername>;Password=<password>;Integrated Security=False;Encrypt=True;Connect Timeout=30"
            }
        }
    }

Στο το παραπάνω JSON: 

- Τύπος έχει οριστεί σε AzureSqlDatabase
- ιδιότητα τύπος συμβολοσειρά σύνδεσης καθορίζει τις πληροφορίες για να συνδεθείτε με μια βάση δεδομένων Azure SQL.  


Όπως μπορείτε να δείτε, την υπηρεσία συνδεδεμένων καθορίζει τον τρόπο σύνδεσης με μια βάση δεδομένων Azure SQL. Το σύνολο δεδομένων καθορίζει ποιος πίνακας χρησιμοποιείται ως μια εισόδου/εξόδου για τη δραστηριότητα σε μια διαδικασία. Στην ενότητα δραστηριότητας σε σας [διοχέτευσης](data-factory-create-pipelines.md) JSON Καθορίζει εάν το σύνολο δεδομένων χρησιμοποιούνται ως ένα σύνολο δεδομένων εισόδου ή εξόδου.


> [AZURE.IMPORTANT] Εκτός εάν ένα σύνολο δεδομένων που παράγεται από προέλευση δεδομένων του Azure, πρέπει να επισημαίνεται ως **εξωτερική**. Αυτή η ρύθμιση ισχύει γενικά για εισροές πρώτη δραστηριότητα σε μια διαδικασία.

## <a name="Structure"></a>Δομή του συνόλου δεδομένων

Στο παρακάτω παράδειγμα, το σύνολο δεδομένων έχει slicetimestamp τρεις στήλες, όνομα_έργου και pageviews και είναι του τύπου: συμβολοσειρά, συμβολοσειρά και δεκαδικών αντίστοιχα.

    structure:  
    [ 
        { "name": "slicetimestamp", "type": "String"},
        { "name": "projectname", "type": "String"},
        { "name": "pageviews", "type": "Decimal"}
    ]

## <a name="Availability"></a>Διαθεσιμότητα συνόλου δεδομένων
Στην ενότητα **διαθεσιμότητα** σε ένα σύνολο δεδομένων ορίζει το παράθυρο επεξεργασίας (ανά ώρα, ημερήσια, εβδομαδιαία, κ.λπ.) ή το μοντέλο slicing για το σύνολο δεδομένων. Ανατρέξτε στο θέμα [Προγραμματισμός και εκτέλεσης](data-factory-scheduling-and-execution.md) άρθρο για περισσότερες λεπτομέρειες σχετικά με το μοντέλο σε φέτες και εξάρτησης του συνόλου δεδομένων. 

Στην παρακάτω ενότητα διαθεσιμότητα Καθορίζει ότι το σύνολο δεδομένων εξόδου που παράγονται ανά ώρα (ή είναι) εισαγωγής συνόλου δεδομένων είναι διαθέσιμη η ώρα:

    "availability": 
    {   
        "frequency": "Hour",        
        "interval": 1   
    }

Ο παρακάτω πίνακας περιγράφει τις ιδιότητες που μπορείτε να χρησιμοποιήσετε στην ενότητα διαθεσιμότητα: 

| Ιδιότητα | Περιγραφή | Απαιτείται | Προεπιλεγμένη |
| -------- | ----------- | -------- | ------- |
| συχνότητα | Η προεπιλεγμένη μονάδα χρόνου για την παραγωγή φέτα συνόλου δεδομένων.<br/><br/>**Υποστηριζόμενα συχνότητα**: λεπτό, ώρα, ημέρα, εβδομάδα, μήνας | Ναι | NA |
| χρονικό διάστημα | Καθορίζει μια τραπεζιού για συχνότητα<br/><br/>"Διάστημα συχνότητας x" Καθορίζει πόσο συχνά παράγεται στη φέτα.<br/><br/>Εάν χρειάζεστε το σύνολο δεδομένων για να είναι τεμαχισμένη σε ωριαία βάση, μπορείτε να ορίσετε **συχνότητα** να **ώρα**και **διάστημα** σε **1**.<br/><br/>**Σημείωση:** Εάν καθορίσετε συχνότητα ως λεπτό, συνιστάται να ορίσετε το χρονικό διάστημα δεν μπορεί να είναι μικρότερη από 15 | Ναι | NA |
| στυλ | Καθορίζει αν θα πρέπει να παραχθεί στη φέτα κατά την έναρξη/λήξη του χρονικού διαστήματος.<ul><li>StartOfInterval</li><li>EndOfInterval</li></ul><br/><br/>Εάν έχει οριστεί συχνότητα μήνα και στυλ έχει οριστεί σε EndOfInterval, δημιουργείται στη φέτα την τελευταία ημέρα του μήνα. Εάν το στυλ που έχει οριστεί σε StartOfInterval, δημιουργείται στη φέτα την πρώτη ημέρα του μήνα.<br/><br/>Εάν έχει οριστεί συχνότητα ημέρα και στυλ έχει οριστεί σε EndOfInterval, δημιουργείται στη φέτα την τελευταία ώρα της ημέρας.<br/><br/>Εάν έχει οριστεί συχνότητα σε ώρα και στυλ έχει οριστεί σε EndOfInterval, τη φέτα δημιουργείται στο τέλος της ώρας. Για παράδειγμα, για ένα κομμάτι για περίοδο μμ 1-2 μμ, τη φέτα παράγεται στις 2 μμ. | Όχι | EndOfInterval |
| anchorDateTime | Καθορίζει την απόλυτη θέση σε χρόνο που χρησιμοποιείται από το χρονοδιάγραμμα για τον υπολογισμό του συνόλου δεδομένων φέτα όρια. <br/><br/>**Σημείωση:** Εάν το AnchorDateTime έχει τμήματα ημερομηνίας που είναι πιο λεπτομερές από τη συχνότητα παραβλέπονται τα πιο λεπτομερές τμήματα. <br/><br/>Για παράδειγμα, εάν το **διάστημα** είναι η **ώρα** (συχνότητα: ώρα και διάστημα: 1) και το **AnchorDateTime** περιέχει **λεπτά και δευτερόλεπτα**και, στη συνέχεια, παραβλέπονται τα μέρη του AnchorDateTime **λεπτά και δευτερόλεπτα** . | Όχι | 01/01/0001 |
| Μετατόπιση | Χρονικό διάστημα κατά το οποίο μετακινούνται την έναρξη και λήξη όλες τις φέτες συνόλου δεδομένων. <br/><br/>**Σημείωση:** Εάν έχουν καθοριστεί anchorDateTime και μετατόπιση, το αποτέλεσμα είναι το συνδυασμένο shift. | Όχι | NA |

### <a name="offset-example"></a>παράδειγμα OFFSET

Ημερήσια χωρίζει που ξεκινούν στις 6 ΠΜ αντί για την προεπιλεγμένη τα μεσάνυχτα. 

    "availability":
    {
        "frequency": "Day",
        "interval": 1,
        "offset": "06:00:00"
    }

Η **συχνότητα** έχει οριστεί σε **ημέρα** και **διάστημα** έχει οριστεί σε **1** (μία φορά την ημέρα): Εάν θέλετε στη φέτα που θα παραχθεί στις 6 ΠΜ αντί για την προεπιλεγμένη ώρα: 12 ΠΜ. Να θυμάστε ότι αυτή η ώρα είναι μια ώρα UTC. 

## <a name="anchordatetime-example"></a>παράδειγμα anchorDateTime

**Παράδειγμα:** 23 ώρες dataset φέτες που ξεκινούν την 2007-04-19T08:00:00

    "availability": 
    {   
        "frequency": "Hour",        
        "interval": 23, 
        "anchorDateTime":"2007-04-19T08:00:00"  
    }

## <a name="offsetstyle-example"></a>Παράδειγμα OFFSET/στυλ

Εάν χρειάζεστε σύνολο δεδομένων σε μηνιαία βάση σε συγκεκριμένη ημερομηνία και ώρα (ας υποθέσουμε ότι στην 3ος κάθε μήνα στις 8:00 πμ), χρήση θα πρέπει να εκτελέσετε την ετικέτα **offset** για να ορίσετε την ημερομηνία και την ώρα. 

    {
      "name": "MyDataset",
      "properties": {
        "type": "AzureSqlTable",
        "linkedServiceName": "AzureSqlLinkedService",
        "typeProperties": {
          "tableName": "MyTable"
        },
        "availability": {
          "frequency": "Month",
          "interval": 1,
          "offset": "3.08:10:00",
          "style": "StartOfInterval"
        }
      }
    }


## <a name="Policy"></a>Πολιτική συνόλου δεδομένων

Στην ενότητα **πολιτική** στον ορισμό του συνόλου δεδομένων καθορίζει τα κριτήρια ή τη συνθήκη που πρέπει να ικανοποιεί τις φέτες του συνόλου δεδομένων.

### <a name="validation-policies"></a>Πολιτικές επικύρωσης

| Όνομα πολιτικής | Περιγραφή | Εφαρμόζεται σε | Απαιτείται | Προεπιλεγμένη |
| ----------- | ----------- | ---------- | -------- | ------- |
| minimumSizeMB | Επαληθεύει ότι τα δεδομένα σε ένα **αντικειμένων blob του Azure** πληροί τις απαιτήσεις ελάχιστο μέγεθος (σε megabyte). | Αντικειμένων Blob του Azure | Όχι | NA |
|minimumRows | Επαληθεύει ότι τα δεδομένα σε μια **βάση δεδομένων Azure SQL** ή του **Azure πίνακας** περιέχει τον ελάχιστο αριθμό γραμμών. | <ul><li>Βάση δεδομένων SQL Azure</li><li>Πινάκων του Azure</li></ul> | Όχι | NA

#### <a name="examples"></a>Παραδείγματα

**minimumSizeMB:**

    "policy":
    
    {
        "validation":
        {
            "minimumSizeMB": 10.0
        }
    }

**minimumRows**

    "policy":
    {
        "validation":
        {
            "minimumRows": 100
        }
    }

### <a name="external-datasets"></a>Σύνολα εξωτερικών δεδομένων

Εξωτερική συνόλων δεδομένων είναι εκείνα που δεν παράγονται από μια ενσωματωμένη διοχέτευσης στο την προέλευση δεδομένων. Εάν το σύνολο δεδομένων έχει επισημανθεί ως **εξωτερική**, την πολιτική **ExternalData** μπορεί να οριστεί να επηρεάσει τη συμπεριφορά της διαθεσιμότητας φέτα συνόλου δεδομένων. 

Εκτός εάν ένα σύνολο δεδομένων που παράγεται από προέλευση δεδομένων του Azure, πρέπει να επισημαίνεται ως **εξωτερική**. Αυτή η ρύθμιση ισχύει για τα δεδομένα εισόδου της πρώτης δραστηριότητας σε μια διαδικασία Γενικά, εκτός εάν δραστηριότητα ή αλυσιδωτή σύνδεση δικτύου αγωγών που χρησιμοποιείται. 

| Όνομα | Περιγραφή | Απαιτείται | Προεπιλεγμένη τιμή  |
| ---- | ----------- | -------- | -------------- |
| dataDelay | Χρόνος για να καθυστερήσετε την έλεγχος τη διαθεσιμότητα των εξωτερικών δεδομένων για το συγκεκριμένο κομμάτι. Για παράδειγμα, εάν τα δεδομένα είναι πρέπει να είναι διαθέσιμη ανά ώρα, ο έλεγχος για να δείτε τα εξωτερικά δεδομένα είναι διαθέσιμη και το αντίστοιχο φέτα είναι έτοιμο μπορεί να καθυστερήσει με τη χρήση dataDelay.<br/><br/>Ισχύει μόνο για εκείνη τη στιγμή.  Για παράδειγμα, εάν είναι 1:00 μμ αυτήν τη στιγμή και αυτή η τιμή είναι 10 λεπτά, η επικύρωση ξεκινά από 1:10 Μ.Μ.<br/><br/>Αυτή η ρύθμιση δεν επηρεάζει φέτες στο παρελθόν (φέτες με ώρα λήξης φέτα + dataDelay < τώρα) πραγματοποιείται χωρίς καθυστέρηση.<br/><br/>Μεγαλύτερη από 23:59 ώρες πρέπει να καθοριστεί χρησιμοποιώντας τη μορφή day.hours:minutes:seconds ώρα. Για παράδειγμα, για να καθορίσετε τις 24 ώρες, μην χρησιμοποιήσετε 24:00:00 Εναλλακτικά, χρησιμοποιήστε 1.00:00:00. Εάν χρησιμοποιείτε το 24:00:00, εκλαμβάνεται ως 24 ημέρες (24.00:00:00). Για 1 ημέρα και 4 ώρες, καθορίστε 1:04:00:00. | Όχι | 0 |
