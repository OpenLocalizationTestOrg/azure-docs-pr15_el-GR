# <a name="replicate-on-premises-vmware-virtual-machines-or-physical-servers-to-a-secondary-site"></a>Αναπαραγωγή εσωτερικής VMware εικονικές μηχανές ή φυσικής διακομιστές σε μια δευτερεύουσα τοποθεσία


## <a name="overview"></a>Επισκόπηση

InMage Scout στο Azure τοποθεσίας αποκατάστασης παρέχει σε πραγματικό χρόνο αναπαραγωγή μεταξύ τοποθεσιών VMware εσωτερικής εγκατάστασης. InMage Scout περιλαμβάνεται σε συνδρομές υπηρεσίας ανάκτησης Azure τοποθεσίας.


## <a name="prerequisites"></a>Προαπαιτούμενα στοιχεία

**Λογαριασμός Azure**: θα χρειαστείτε ένα λογαριασμό [Microsoft Azure](https://azure.microsoft.com/) . Μπορείτε να ξεκινήσετε με μια [δωρεάν δοκιμαστική έκδοση](https://azure.microsoft.com/pricing/free-trial/). [Μάθετε περισσότερα](https://azure.microsoft.com/pricing/details/site-recovery/) σχετικά με τις τιμές Επαναφορά τοποθεσίας.


## <a name="step-1-create-a-vault"></a>Βήμα 1: Δημιουργήστε ένα θάλαμο
1. Είσοδος στην [πύλη του Azure](https://portal.azure.com).
2. Κάντε κλικ στην επιλογή νέα > Διαχείριση > Δημιουργία αντιγράφων ασφαλείας και Επαναφορά τοποθεσίας (OMS). Εναλλακτικά, μπορείτε να κάνετε κλικ αναζήτηση > θάλαμο υπηρεσίες ανάκτησης > Προσθήκη.
3. Στο πλαίσιο **όνομα** , καθορίστε ένα φιλικό όνομα για τον προσδιορισμό του θάλαμο. Εάν έχετε περισσότερες από μία συνδρομές, επιλέξτε μία από αυτές.
4. Στην **ομάδα πόρων** , δημιουργήστε μια νέα ομάδα πόρων ή επιλέξτε ένα υπάρχον. Καθορίστε μια περιοχή Azure για να ολοκληρώσετε τα απαιτούμενα πεδία. 
5.  Στην **τοποθεσία**, επιλέξτε τη γεωγραφική περιοχή για το θάλαμο. Για να ελέγξετε την υποστηριζόμενη περιοχές, ανατρέξτε στο θέμα [Τις τιμές του Azure τοποθεσίας αποκατάστασης](https://azure.microsoft.com/pricing/details/site-recovery/).
5. Εάν θέλετε να γρήγορα πρόσβαση το θάλαμο από τον πίνακα εργαλείων, κάντε κλικ στην επιλογή Καρφίτσωμα στον πίνακα εργαλείων και, στη συνέχεια, κάντε κλικ στην επιλογή Δημιουργία.
6. Θα εμφανιστεί η νέα θάλαμο στον πίνακα εργαλείων > όλους τους πόρους, και σχετικά με τις υπηρεσίες ανάκτησης κύριο χώροι φύλαξης blade.

## <a name="step-2-configure-the-vault-and-download-inmage-scout-components"></a>Βήμα 2: Ρύθμιση παραμέτρων του θάλαμο και λήψη InMage Scout στοιχείων
7. Στις υπηρεσίες ανάκτησης blade χώροι φύλαξης επιλέξτε θάλαμο σας και κάντε κλικ στην επιλογή ρυθμίσεις.
8. Στις **Ρυθμίσεις** > **Γρήγορα αποτελέσματα** κάντε κλικ στην επιλογή **Επαναφορά τοποθεσίας** > βήμα 1: **Προετοιμασία υποδομή** > **στόχος προστασίας**.
9. Στο **στόχο προστασίας** επιλέξτε αποκατάστασης τοποθεσία και επιλέξτε Ναι, με vSphere VMware υπερεπόπτη. Στη συνέχεια, κάντε κλικ στο κουμπί OK.
10. Στο **πρόγραμμα εγκατάστασης Scout**, κάντε κλικ στην επιλογή λήψη για να κάνετε λήψη InMage Scout 8.0.1 GA λογισμικού και καταχώρηση αριθμού-κλειδιού. Τα αρχεία εγκατάστασης για όλα τα απαιτούμενα στοιχεία που υπάρχουν στο αρχείο .zip που έχετε λάβει.


## <a name="step-3-install-component-updates"></a>Βήμα 3: Εγκατάσταση των ενημερώσεων του στοιχείου

Διαβάστε σχετικά με τις πιο πρόσφατες [ενημερώσεις](#updates). Θα μπορείτε να εγκαταστήσετε την ενημέρωση αρχείων σε διακομιστές με την εξής σειρά:

1. Διακομιστή ΛΉΨΗΣ, εάν υπάρχει ένα
2. Ρύθμιση παραμέτρων διακομιστών
3. Οι διακομιστές διαδικασία
3. Διακομιστές κύρια προορισμού
4. Οι διακομιστές vContinuum
5. Διακομιστή προέλευσης (Windows και Linux Server)

Εγκαταστήστε τις ενημερώσεις ως εξής:

1. Κάντε λήψη του αρχείου .zip [Ενημέρωση](https://aka.ms/asr-scout-update4) . Αυτό το αρχείο .zip περιέχει τα ακόλουθα αρχεία:

    - RX_8.0.4.0_GA_Update_4_8725872_16Sep16.Tar.GZ
    - CX_Windows_8.0.4.0_GA_Update_4_8725865_14Sep16.exe
    - UA_Windows_8.0.4.0_GA_Update_4_9035261_27Sep16.exe
    - UA_RHEL6 64_8.0.4.0_GA_Update_4_9035261_26Sep16.tar.gz
    - vCon_Windows_8.0.4.0_GA_Update_4_8921562_16Sep16.exe
    - Bit update4 UA για RHEL5, OL5, OL6, SUSE 10, SUSE 11: UA_<Linux OS>_8.0.4.0_GA_Update_4_9035261_26Sep16.tar.gz 
    
2. Εξαγάγετε τα αρχεία .zip.<br>
3. **Για ΛΉΨΗΣ διακομιστή**: Αντιγραφή **RX_8.0.4.0_GA_Update_4_8725872_16Sep16.tar.gz** στο διακομιστή ΛΉΨΗΣ και εξαγάγετε. Στο φάκελο που έχει εξαχθεί, **εκτελέστε/εγκατάσταση**.<br>
4. **Για το διακομιστή ρύθμισης παραμέτρων διακομιστή/διαδικασία**: Αντιγραφή **CX_Windows_8.0.4.0_GA_Update_4_8725865_14Sep16.exe** στο διακομιστή ρύθμισης παραμέτρων και διαδικασία διακομιστή. Κάντε διπλό κλικ για να το εκτελέσετε.<br>
5. **Οι κύριες για Windows server προορισμού**: για να ενημερώσετε τον παράγοντα ενοποιημένου, αντιγράψτε **UA_Windows_8.0.4.0_GA_Update_4_9035261_27Sep16.exe** στο διακομιστή κύρια προορισμού. Κάντε διπλό κλικ για να το εκτελέσετε. Σημειώστε ότι ο παράγοντας ενοποιημένου ισχύει επίσης στο διακομιστή προέλευσης. Που θα πρέπει να εγκαταστήσετε στο διακομιστή προέλευσης καθώς και, όπως αναφέρθηκε αργότερα σε αυτήν τη λίστα.<br>
7. **Για το διακομιστή vContinuum**: Αντιγραφή **vCon_Windows_8.0.4.0_GA_Update_4_8921562_16Sep16.exe** στο διακομιστή vContinuum.  Βεβαιωθείτε ότι έχετε κλείσει τον "Οδηγό vContinuum". Κάντε διπλό κλικ στο αρχείο για να το εκτελέσετε.<br>
8. **Διακομιστής κύριας προορισμού για Linux**: για να ενημερώσετε τον παράγοντα ενοποιημένου, αντιγράψτε **UA_RHEL6 64_8.0.4.0_GA_Update_4_9035261_26Sep16.tar.gz** στο διακομιστή κύρια προορισμού και εξαγάγετε. Στο φάκελο που έχει εξαχθεί, **εκτελέστε/εγκατάσταση**.<br>
9. **Για Windows προέλευσης διακομιστή**: για να ενημερώσετε τον παράγοντα ενοποιημένου, αντιγράψτε **UA_Windows_8.0.4.0_GA_Update_4_9035261_27Sep16.exe** στο διακομιστή προέλευσης. Κάντε διπλό κλικ για να το εκτελέσετε.<br>
10. **Διακομιστή προέλευσης για Linux**: για να ενημερώσετε τον παράγοντα ενοποιημένος χώρος, αντιγράψτε αντίστοιχη έκδοση του αρχείου UA στο διακομιστή Linux και εξαγάγετε. Στο φάκελο που έχει εξαχθεί, **εκτελέστε/εγκατάσταση**.  Παράδειγμα: RHEL διακομιστής 6,7 64 bit, αντιγράψτε **UA_RHEL6 64_8.0.4.0_GA_Update_4_9035261_26Sep16.tar.gz** στο διακομιστή και την εξαγωγή. Στο φάκελο που έχει εξαχθεί, **εκτελέστε/εγκατάσταση**.

### <a name="azure-site-recovery-scout-801-update-3"></a>Ενημέρωση Scout 8.0.1 ανάκτησης Azure τοποθεσίας 3
3 η ενημερωμένη έκδοση περιλαμβάνει τις ακόλουθες διορθώσεις σφαλμάτων και βελτιώσεις:

- Ο διακομιστής ρύθμισης παραμέτρων και ΛΉΨΗΣ αποτύχει για την καταχώρηση για το θάλαμο Επαναφορά τοποθεσίας όταν βρίσκονται πίσω από το διακομιστή μεσολάβησης.
- Ο αριθμός των ωρών που δεν ικανοποιείται το στόχο σημείου αποκατάστασης (RPO) δεν γρήγορα ενημερώνεται η αναφορά εύρυθμης λειτουργίας.
- Ο διακομιστής ρύθμισης παραμέτρων δεν συγχρονίζεται με ΛΉΨΗΣ όταν το ESX λεπτομέρειες υλικού ή λεπτομέρειες δικτύου περιέχει χαρακτήρες UTF-8.
- Windows Server 2008 R2 ελεγκτές αποτύχει για εκκίνηση μετά την ανάκτηση.
- Συγχρονισμού χωρίς σύνδεση δεν λειτουργεί όπως αναμένεται.
- Μετά την ανακατεύθυνση εικονική μηχανή (Εικονική), διαγραφή αναπαραγωγής ζεύγος παραμένει στο περιβάλλον εργασίας Χρήστη του CX για μεγάλο χρονικό διάστημα, και οι χρήστες δεν είναι δυνατό να ολοκληρώσετε την αποκατάσταση μετά από ή να συνεχίσετε τη λειτουργία.
- Συνολική στιγμιότυπο λειτουργίες που εκτελούνται από το έργο συνέπειας έχουν βελτιστοποιηθεί για τη μείωση της εφαρμογής αποσυνδέει όπως προγράμματα-πελάτες του SQL.
- Οι επιδόσεις του εργαλείου συνέπειας (VACP.exe) έχουν βελτιωθεί, μειώνοντας τη χρήση μνήμης που απαιτείται για τη δημιουργία στιγμιότυπα στα Windows.
- Το πάτημα εγκατάσταση υπηρεσίας παρουσιάσει σφάλμα όταν ο κωδικός πρόσβασης είναι μεγαλύτερο από 16 χαρακτήρες.
- vContinuum δεν είναι ο έλεγχος και να γίνεται ερώτηση για τη νέα διαπιστευτήρια vCenter όταν αλλάζουν τα διαπιστευτήρια.
- Στην Linux, η διαχείριση cache κύρια προορισμού (cachemgr) δεν γίνεται λήψη αρχείων από το διακομιστή διαδικασία, με αποτέλεσμα την αναπαραγωγή ζεύγος περιορισμού.
- Όταν η σειρά φυσικής ανακατεύθυνσης συμπλέγματος (MSCS) δίσκου δεν είναι η ίδια σε όλους τους κόμβους, η αναπαραγωγή δεν έχει οριστεί για ορισμένα από τα όγκους σύμπλεγμα.
<br/>Σημειώστε ότι το σύμπλεγμα πρέπει να είναι reprotected για να επωφεληθείτε από αυτήν την ενημέρωση κώδικα.  
- Η λειτουργία SMTP δεν λειτουργεί όπως αναμένεται μετά την αναβάθμιση της ΛΉΨΗΣ από Scout 7.1 σε Scout 8.0.1.

### <a name="azure-site-recovery-scout-801-update-2-update-03dec15"></a>Ενημέρωση Scout 8.0.1 ανάκτησης Azure τοποθεσία 2 (ενημέρωση 03 15 Δεκ)

Επιδιορθώσεις στο ενημερωμένη έκδοση 2 περιλαμβάνουν τα εξής:

- **Ρύθμιση παραμέτρων διακομιστή**: διόρθωση για ένα θέμα που εμπόδισε τη δυνατότητα δωρεάν μέτρησης 31-ημέρα λειτουργεί όπως αναμένεται όταν ο διακομιστής ρύθμισης παραμέτρων έχει καταχωρηθεί στην τοποθεσία ανάκτηση.
- **Παράγοντας ενοποιημένης**: διόρθωση για ένα θέμα στο 1 ενημέρωση που οδήγησε σε την ενημέρωση δεν εγκαθίσταται στο διακομιστή κύρια προορισμού όταν έγινε αναβάθμιση από την έκδοση 8.0 να 8.0.1.


### <a name="azure-site-recovery-scout-801-update-1"></a>Ενημέρωση Scout 8.0.1 ανάκτησης Azure τοποθεσία 1

Ενημέρωση 1 περιλαμβάνει τα παρακάτω διορθώσεις σφαλμάτων και τις νέες δυνατότητες:

- 31 ημέρες δωρεάν προστασίας ανά παρουσία διακομιστή. Αυτό σας επιτρέπει να δοκιμάσετε τη λειτουργικότητα του ή να ρυθμίσετε μια απόδειξη λειτουργίας.
    - Όλες οι λειτουργίες στο διακομιστή, συμπεριλαμβανομένων των ανακατεύθυνσης και επιστροφή σε αυτόν, είναι δωρεάν για τις πρώτες 31 ημέρες, ξεκινώντας από τη στιγμή που ένα διακομιστή πρώτα προστατεύεται με Scout ανάκτησης τοποθεσίας.
    - Από την 32ης ημέρα και μετά, κάθε προστατευμένο διακομιστή θα χρεωθεί με το επιτόκιο τυπική παρουσία προστασίας Azure Επαναφορά τοποθεσίας σε μια τοποθεσία που ανήκουν στον πελάτη.
    - Οποιαδήποτε στιγμή, τον αριθμό των προστατευμένων διακομιστές που χρεώνεστε αυτήν τη στιγμή είναι διαθέσιμη στη σελίδα πίνακα εργαλείων της το θάλαμο Azure τοποθεσίας αποκατάστασης.
- Υποστήριξη προστεθεί για vSphere περιβάλλον γραμμής εντολών (vCLI) 5,5 ενημερωμένη έκδοση 2.
- Υποστήριξη που προστέθηκε για λειτουργικά συστήματα Linux στο διακομιστή προέλευσης:
    - RHEL 6 ενημέρωση 6
    - RHEL 5 ενημέρωση 11
    - Ενημέρωση centOS 6 6
    - Ενημέρωση centOS 5 11
- Σφάλμα διορθώνει να αντιμετωπίζει τα ακόλουθα θέματα:
    - Καταχώρηση θάλαμο αποτυγχάνει για τη ρύθμιση παραμέτρων διακομιστή ή το διακομιστή ΛΉΨΗΣ.
    - Σύμπλεγμα όγκους δεν εμφανίζεται όπως αναμένεται όταν ομαδοποιημένων εικονικές μηχανές είναι reprotected όταν αυτά βιογραφικό σημείωμα.
    - Αποκατάσταση μετά από αποτυγχάνει, όταν ο διακομιστής κύριας προορισμού φιλοξενείται σε διαφορετικό διακομιστή ESXi από τις εικονικές μηχανές παραγωγής εσωτερικής εγκατάστασης.
    - Ρύθμιση παραμέτρων δικαιωμάτων αρχείου αλλάζουν κατά την αναβάθμιση σε 8.0.1, η οποία επηρεάζει την προστασία και λειτουργίες.
    - Το όριο επανάληψη του συγχρονισμού δεν ισχύ με τον αναμενόμενο τρόπο, που οδηγεί σε συμπεριφορά χωρίς συνέπεια αναπαραγωγής.
    - Οι ρυθμίσεις RPO δεν εμφανίζονται σωστά στο περιβάλλον εργασίας του διακομιστή ρύθμισης παραμέτρων. Η τιμή αρχεία μη συμπιεσμένων δεδομένων εμφανίζει εσφαλμένα την συμπιεσμένο τιμή.
    -  Η λειτουργία κατάργηση δεν διαγράφει όπως αναμένεται στον Οδηγό vContinuum και αναπαραγωγής δεν διαγράφονται από το περιβάλλον εργασίας του διακομιστή ρύθμισης παραμέτρων.
    -  Στον Οδηγό vContinuum, ο δίσκος είναι δεν είναι επιλεγμένο αυτόματα όταν κάνετε κλικ στην προβολή δίσκου **Λεπτομέρειες** κατά τη διάρκεια της προστασίας του MSCS εικονικές μηχανές.
    - Κατά τη διάρκεια του φυσικά-σε-εικονικού σενάριο (P2V), απαιτείται HP υπηρεσίες, όπως CIMnotify και CqMgHost, δεν είναι μετακινηθεί σε μη αυτόματα στο εικονική μηχανή αποκατάστασης. Το αποτέλεσμα χρόνο επιπλέον εκκίνησης.
    - Προστασία εικονική μηχανή Linux αποτυγχάνει όταν υπάρχουν περισσότερες από 26 δίσκων στο διακομιστή κύρια προορισμού.
