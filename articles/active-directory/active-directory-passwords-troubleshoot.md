## <a name="information-to-include-when-you-need-help"></a>Πληροφορίες που θα συμπεριληφθούν όταν χρειάζεστε βοήθεια

Εάν δεν μπορείτε να επιλύσετε το ζήτημα με τις παρακάτω οδηγίες, μπορείτε να επικοινωνήσετε μας μηχανικοί υποστήριξης. Όταν επικοινωνείτε με τους, συνιστάται να συμπεριλάβετε τις παρακάτω πληροφορίες:

 - **Γενική περιγραφή του σφάλματος** – τι ακριβές μήνυμα λάθους μήπως το χρήστη να δείτε;  Εάν υπήρχε κανένα μήνυμα σφάλματος, Περιγράψτε το μη αναμενόμενη συμπεριφορά διαπιστώσατε, με λεπτομέρειες.
 - **Σελίδα** – τι σελίδας που έχουν στην όταν είδατε το σφάλμα (συμπεριλάβετε τη διεύθυνση URL);
 - **Ημερομηνίας / ώρας / ζώνης ώρας** – τι έχει την ακριβή ημερομηνία και ώρα που είδατε το σφάλμα (περιλαμβάνει τη ζώνη ώρας);
 - **Υποστήριξη κώδικα** – τι ήταν ο κώδικας υποστήριξης που δημιουργείται όταν ο χρήστης είδατε το σφάλμα (για να βρείτε αυτό, η αναπαραγωγή του σφάλματος, στη συνέχεια, κάντε κλικ στη σύνδεση υποστηρίζει κώδικα στο κάτω μέρος της οθόνης και αποστολή του μηχανικό υποστήριξης το GUID που προκύπτει).
   - Εάν είστε σε μια σελίδα χωρίς κωδικό υποστήριξης στο κάτω μέρος, πατήστε το πλήκτρο F12 και αναζήτησης για το αναγνωριστικό ΑΣΦΑΛΕΊΑΣ και CID και στείλτε αυτές τις δύο αποτελεσμάτων για το μηχανικό υποστήριξης.

    ![][001]

 - **Αναγνωριστικό χρήστη** – ποιο ήταν το Αναγνωριστικό του χρήστη που είδατε το σφάλμα (π.χ.user@contoso.com)?
 - **Πληροφορίες σχετικά με το χρήστη** -ήταν το χρήστη ομόσπονδες, κατακερματισμός τον κωδικό πρόσβασης που συγχρονίσατε, μόνο cloud;  Ο χρήστης έχει μια AAD Premium ή βασικές AAD εκχωρηθεί άδεια χρήσης;
 - **Αρχείο καταγραφής συμβάντων εφαρμογής** – Εάν χρησιμοποιείτε αντιγραφής τον κωδικό πρόσβασης και το σφάλμα είναι στην υποδομή σας εσωτερικής εγκατάστασης, ανατρέξτε zip ένα αντίγραφο των το αρχείο καταγραφής συμβάντων της εφαρμογής από το διακομιστή σας Azure AD Connect και αποστολή μαζί με την αίτησή σας.

Συμπεριλάβετε τις παρακάτω πληροφορίες θα σας βοηθήσει να επιλύσετε το πρόβλημα συντομότερο δυνατό.

## <a name="troubleshoot-password-writeback-connectivity"></a>Αντιμετώπιση προβλημάτων συνδεσιμότητας αντιγραφής κωδικού πρόσβασης

Εάν αντιμετωπίζετε διακοπές υπηρεσίας με το στοιχείο αντιγραφής τον κωδικό πρόσβασης του Azure AD Connect, εδώ θα βρείτε ορισμένες γρήγορα βήματα που μπορείτε να ακολουθήσετε για να επιλύσετε αυτό το θέμα:

 - [Επανεκκίνηση του Azure AD σύνδεση συγχρονισμού υπηρεσίας](#restart-the-azure-AD-Connect-sync-service)
 - [Απενεργοποίηση και να ενεργοποιήσετε ξανά τη δυνατότητα αντιγραφής κωδικού πρόσβασης](#disable-and-re-enable-the-password-writeback-feature)
 - [Εγκαταστήστε την πιο πρόσφατη έκδοση του Azure AD Connect](#install-the-latest-azure-ad-connect-release)
 - [Αντιμετώπιση προβλημάτων αντιγραφής κωδικού πρόσβασης](#troubleshoot-password-writeback)

Σε γενικές γραμμές, συνιστάται να εκτελέσετε τα παρακάτω βήματα με τη σειρά παραπάνω για να ανακτήσετε την υπηρεσία κατά τον ταχύτερο τρόπο.

### <a name="restart-the-azure-ad-connect-sync-service"></a>Επανεκκίνηση του Azure AD σύνδεση συγχρονισμού υπηρεσίας
Επανεκκίνηση της Azure AD σύνδεση συγχρονισμού υπηρεσίας μπορούν να σας βοηθήσουν να επιλύσετε προβλήματα συνδεσιμότητας ή άλλα μεταβατικές προβλήματα με την υπηρεσία.

 1. Ως διαχειριστής, κάντε κλικ στην επιλογή **Έναρξη** στο διακομιστή που εκτελεί **Azure AD Connect**.
 2. Πληκτρολογήστε **"services.msc"** στο πλαίσιο αναζήτησης και πατήστε το πλήκτρο **Enter**.
 3. Αναζητήστε το **Microsoft Azure AD Connect** καταχώρηση.
 4. Κάντε δεξί κλικ στην εγγραφή υπηρεσίας, κάντε κλικ στην επιλογή **επανεκκίνηση**και περιμένετε να ολοκληρωθεί η λειτουργία.

    ![][002]

Τα παρακάτω βήματα θα αποκαταστήσετε τη σύνδεση με την υπηρεσία cloud και επίλυση ΤΥΧΟΝ ΔΙΑΚΟΠΕΣ ενδέχεται να αντιμετωπίζετε.  Αν η επανεκκίνηση της υπηρεσίας συγχρονισμού δεν έλυσε το ζήτημά σας, συνιστάται να δοκιμάσετε να απενεργοποιήσετε και ενεργοποιήσετε ξανά τη δυνατότητα αντιγραφής τον κωδικό πρόσβασης ως επόμενο βήμα.

### <a name="disable-and-re-enable-the-password-writeback-feature"></a>Απενεργοποίηση και να ενεργοποιήσετε ξανά τη δυνατότητα αντιγραφής κωδικού πρόσβασης
Απενεργοποιήσετε και ενεργοποιήσετε ξανά τη δυνατότητα αντιγραφής τον κωδικό πρόσβασης μπορούν να σας βοηθήσουν να επιλύσετε προβλήματα συνδεσιμότητας.

 1. Ως διαχειριστής, ανοίξτε τον **Οδηγό ρύθμισης παραμέτρων Azure AD Connect**.
 2. Στο παράθυρο διαλόγου **σύνδεση με το Azure AD** , εισαγάγετε τα **διαπιστευτήρια Azure AD καθολικός διαχειριστής**
 3. Στο παράθυρο διαλόγου **σύνδεση στο AD DS** , εισαγάγετε τα **διαπιστευτήρια διαχειριστή υπηρεσίες τομέα AD**.
 4. Στο παράθυρο διαλόγου **προσδιορίζει μοναδικά τους χρήστες σας** , κάντε κλικ στο κουμπί **Επόμενο** .
 5. Στο παράθυρο διαλόγου **προαιρετικές δυνατότητες** , καταργήστε την επιλογή από το πλαίσιο ελέγχου **write-back τον κωδικό πρόσβασης** .

    ![][003]

 6. Κάντε κλικ στο κουμπί **Επόμενο** έως τις υπόλοιπες σελίδες στο παράθυρο διαλόγου χωρίς να αλλάξετε οτιδήποτε μέχρι να φτάσετε στη σελίδα **έτοιμο για ρύθμιση παραμέτρων** .
 7. Βεβαιωθείτε ότι η σελίδα Ρύθμιση παραμέτρων εμφανίζει την **επιλογή write-back τον κωδικό πρόσβασης ως απενεργοποιημένη** και, στη συνέχεια, κάντε κλικ στο πράσινο κουμπί **Ρύθμιση παραμέτρων** για να δεσμεύσετε τις αλλαγές σας.
 8. Στο παράθυρο διαλόγου **Ολοκληρωμένη** , καταργήστε την επιλογή **Συγχρονισμός τώρα** και, στη συνέχεια, κάντε κλικ στο κουμπί **Τέλος** για να κλείσετε τον οδηγό.
 9. Εκ νέου άνοιγμα του **Οδηγού ρύθμισης παραμέτρων Azure AD Connect**.
 10.    **Επαναλάβετε τα βήματα 2-8**, εκτός από το βήμα βεβαιωθείτε ότι **ενεργοποιήσετε την επιλογή write-back τον κωδικό πρόσβασης** για τις **προαιρετικές δυνατότητες** οθόνης για να ενεργοποιήσετε ξανά την υπηρεσία.

    ![][004]

Τα παρακάτω βήματα θα αποκαταστήσετε τη σύνδεση με την υπηρεσία στο cloud και επίλυση ΤΥΧΟΝ ΔΙΑΚΟΠΕΣ ενδέχεται να αντιμετωπίζετε.

Εάν απενεργοποιήσετε και ενεργοποιήσετε ξανά τη δυνατότητα αντιγραφής κωδικού πρόσβασης δεν έλυσε το ζήτημά σας, συνιστάται να δοκιμάσετε για να εγκαταστήσετε ξανά Azure AD Connect ως επόμενο βήμα.

### <a name="install-the-latest-azure-ad-connect-release"></a>Εγκαταστήστε την πιο πρόσφατη έκδοση του Azure AD Connect
Επαναλάβετε την εγκατάσταση του πακέτου Azure AD Connect επιλύει τυχόν θέματα ρύθμισης παραμέτρων που μπορεί να επηρεάζει τη δυνατότητά σας να είτε σύνδεση με τις υπηρεσίες cloud μας ή για να διαχειριστείτε τους κωδικούς πρόσβασης στο περιβάλλον του τοπικού AD.
Προτείνουμε να εκτελέσετε αυτό το βήμα μόνο αφού επιχειρήσετε τα πρώτα δύο βήματα που περιγράφονται παραπάνω.

 1. Λήψη πιο πρόσφατη έκδοση του Azure AD Connect [εδώ](active-directory-aadconnect.md#install-azure-ad-connect).
 2. Επειδή έχετε εγκαταστήσει ήδη Azure AD Connect, μόνο θα πρέπει να κάνετε αναβάθμιση στην ίδια θέση για να ενημερώσετε την εγκατάσταση του Azure AD Connect στην πιο πρόσφατη έκδοση.
 3. Εκτελέστε το πακέτο που έχουν ληφθεί και ακολουθήστε τα στην οθόνη οδηγίες για να ενημερώσετε τον υπολογιστή σας Azure AD Connect.  Χωρίς επιπλέον μη αυτόματα βήματα είναι απαραίτητα, εκτός αν έχετε προσαρμόσει εκτός του πλαισίου συγχρονισμός κανόνες, οπότε θα πρέπει να **δημιουργήσετε αυτά τα αντίγραφα ασφαλείας πριν προχωρήσετε σε αναβάθμιση και με μη αυτόματο τρόπο αναπτύξετε ξανά τους αφού ολοκληρώσετε**.

Τα παρακάτω βήματα θα αποκαταστήσετε τη σύνδεση με την υπηρεσία στο cloud και επίλυση ΤΥΧΟΝ ΔΙΑΚΟΠΕΣ ενδέχεται να αντιμετωπίζετε.

Εάν εγκαταστήσετε την πιο πρόσφατη έκδοση του ο διακομιστής Azure AD Connect δεν επιλυθεί το ζήτημά σας, συνιστάται να δοκιμάσετε απενεργοποίηση και εκ νέου ενεργοποίηση αντιγραφής τον κωδικό πρόσβασης ως τελικό βήμα μετά την εγκατάσταση την πιο πρόσφατη συγχρονισμού QFE.

Εάν δεν επιλυθεί το ζήτημά σας, στη συνέχεια, συνιστάται να λάβετε μια ματιά στην [Αντιμετώπιση προβλημάτων αντιγραφής τον κωδικό πρόσβασης](#troubleshoot-password-writeback) και το [Azure AD τον κωδικό πρόσβασης διαχείρισης συνήθεις Ερωτήσεις](active-directory-passwords-faq.md) για να δείτε εάν το πρόβλημα μπορεί να αναφέρεται εκεί.
