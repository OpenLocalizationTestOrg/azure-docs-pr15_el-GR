#<a name="high-availability-cross-geographic-ad-fs-deployment-in-azure-with-azure-traffic-manager"></a>Υψηλή διαθεσιμότητα μεταξύ γεωγραφικών AD FS ανάπτυξης στο Azure με τη Διαχείριση κίνηση Azure

[Ανάπτυξη AD FS στο Azure](active-directory-aadconnect-azure-adfs.md) παρέχει οδηγίες βήμα προς βήμα σχετικά με τον τρόπο μπορείτε να αναπτύξετε μια απλή υποδομή AD FS για την εταιρεία σας στο Azure. Σε αυτό το άρθρο παρέχει τα επόμενα βήματα για να δημιουργήσετε μια μεταξύ γεωγραφικών ανάπτυξη του AD FS στο Azure χρησιμοποιώντας τη [Διαχείριση κίνηση Azure](../traffic-manager/traffic-manager-overview.md). Azure κίνηση Manager βοηθά στη δημιουργία ενός γεωγραφικά Άνοιγμα υψηλή διαθεσιμότητα και την υποδομή AD FS υψηλών επιδόσεων για την εταιρεία σας, καθιστώντας χρήση των περιοχή δρομολόγηση μεθόδους που είναι διαθέσιμες στις διαφορετικές ανάγκες από την υποδομή.

Μια ιδιαίτερα διαθέσιμη μεταξύ γεωγραφικών υποδομή AD FS επιτρέπει:

* **Εξάλειψη των μοναδικό σημείο αποτυχίας:** Με δυνατότητες ανακατεύθυνσης του Azure κίνηση Manager, μπορείτε να επιτύχετε μια ιδιαίτερα διαθέσιμη υποδομή AD FS, ακόμα και όταν ένα από τα κέντρα δεδομένων σε ένα τμήμα της υδρογείου μεταβαίνει προς τα κάτω
* **Βελτιωμένες επιδόσεις:** Μπορείτε να χρησιμοποιήσετε την προτεινόμενη ανάπτυξης σε αυτό το άρθρο για να παρέχουν μια υποδομή AD FS υψηλών επιδόσεων που μπορούν να σας βοηθήσουν τους χρήστες να ελέγχουν την ταυτότητα πιο γρήγορα. 

##<a name="design-principles"></a>Αρχές σχεδίασης

![Συνολική σχεδίασης](./media/active-directory-adfs-in-azure-with-azure-traffic-manager/blockdiagram.png)

Το σχεδίαση βασικές αρχές θα είναι ίδια όπως αναφέρονται στο αρχές σχεδίασης στο άρθρο ανάπτυξης AD FS στο Azure. Το παρακάτω διάγραμμα παρουσιάζει μια απλή επέκταση του την ανάπτυξη βασικές σε ένα άλλο γεωγραφική περιοχή. Ακολουθούν ορισμένα σημεία για να λάβετε υπόψη κατά την επέκταση του ανάπτυξη σε νέα γεωγραφική περιοχή

* **Εικονικές δικτύου:** Πρέπει να δημιουργήσετε ένα νέο εικονικό δίκτυο στο τη γεωγραφική περιοχή που θέλετε να αναπτύξετε πρόσθετη υποδομή του AD FS. Στο παρακάτω διάγραμμα βλέπετε Geo1 VNET και Geo2 VNET ως των δύο εικονικού δικτύων σε κάθε γεωγραφική περιοχή.

* **Ελεγκτές τομέα και διακομιστές AD FS στο νέο γεωγραφική VNET:** Συνιστάται να αναπτύξετε τομέα ελεγκτές στη νέα γεωγραφική περιοχή, έτσι ώστε οι διακομιστές AD FS στη νέα περιοχή δεν χρειάζεται να επικοινωνήσετε με έναν ελεγκτή τομέα σε ένα άλλο μακριά δικτύου για να ολοκληρώσετε μια τον έλεγχο ταυτότητας και συνεπώς τη βελτίωση της απόδοσης.

* **Λογαριασμούς χώρου αποθήκευσης:** Οι λογαριασμοί χώρου αποθήκευσης είναι συσχετισμένη με μια περιοχή. Επειδή πρόκειται να αναπτύξετε μηχανές σε μια νέα γεωγραφική περιοχή, θα πρέπει να δημιουργήσετε νέους λογαριασμούς χώρου αποθήκευσης που θα χρησιμοποιηθεί στην περιοχή.  

* **Δικτύου ομάδες ασφαλείας:** Όπως λογαριασμοί χώρου αποθήκευσης, ομάδες ασφαλείας δικτύου που δημιουργήθηκε σε μια περιοχή δεν μπορούν να χρησιμοποιηθούν σε μια άλλη γεωγραφική περιοχή. Γι ' αυτό, θα πρέπει να δημιουργήσετε νέο δίκτυο ομάδες ασφαλείας παρόμοια με εκείνα του πρώτου γεωγραφική περιοχή για INT και DMZ υποδίκτυο στη νέα γεωγραφική περιοχή.

* **DNS ετικετών για τις δημόσιες διευθύνσεις IP:** Azure κίνηση Manager μπορεί να αναφέρεται σε τελικά σημεία ΜΌΝΟ μέσω ετικέτες DNS. Γι ' αυτό, πρέπει να δημιουργήσετε ετικέτες DNS για το εξωτερικό φόρτωσης Balancers των δημόσιων διευθύνσεων IP.

* **Azure Manager κίνηση:** Διαχείριση την κυκλοφορία του Microsoft Azure σάς επιτρέπει να ελέγχετε την κατανομή κίνηση των χρηστών για να σας τελικά σημεία υπηρεσίας που εκτελούνται σε διαφορετικά κέντρα δεδομένων όλο τον κόσμο. Διαχείριση Azure κίνηση λειτουργεί στο επίπεδο DNS. Χρησιμοποιεί αποκρίσεις DNS για να κατευθύνουν κυκλοφορία τελικού χρήστη για να καθολικά κατανεμημένο τελικά σημεία. Προγράμματα-πελάτες, στη συνέχεια, συνδεθείτε με αυτά τα τελικά σημεία απευθείας. Με διαφορετικές επιλογές δρομολόγησης επιδόσεων, Weighted και προτεραιότητα, μπορείτε εύκολα να επιλέξετε την επιλογή Δρομολόγηση καταλληλότερες για τις ανάγκες της εταιρείας σας. 

* **V-net V δικτύου σύνδεση ανάμεσα σε δύο περιοχές:** Δεν χρειάζεται να υπάρχει σύνδεση μεταξύ των εικονικό δικτύων ίδια. Δεδομένου ότι κάθε εικονικού δικτύου έχει πρόσβαση σε ελεγκτές και έχει διακομιστή ADFS και WAP στο ίδιο, μπορούν να λειτουργούν χωρίς οποιαδήποτε συνδεσιμότητας μεταξύ των εικονικού δικτύων σε διαφορετικές περιοχές. 

##<a name="steps-to-integrate-azure-traffic-manager"></a>Βήματα για την ενοποίηση Manager κίνηση Azure

###<a name="deploy-ad-fs-in-the-new-geographical-region"></a>Ανάπτυξη AD FS στη νέα γεωγραφική περιοχή
Ακολουθήστε τα βήματα και τις οδηγίες στην [Ανάπτυξη AD FS στο Azure](active-directory-aadconnect-azure-adfs.md) για να αναπτύξετε την ίδια τοπολογία στη νέα γεωγραφική περιοχή.

###<a name="dns-labels-for-public-ip-addresses-of-the-internet-facing-public-load-balancers"></a>DNS ετικετών για τις δημόσιες διευθύνσεις IP του το Balancers φόρτωσης Internet αντικριστές (δημόσιας)
Όπως προαναφέρθηκε, τη Διαχείριση κίνηση Azure μόνο μπορεί να αναφέρεται σε ετικέτες DNS ως τελικά σημεία και, επομένως, είναι σημαντικό για να δημιουργήσετε ετικέτες DNS για την εξωτερική φόρτωσης Balancers των δημόσιων διευθύνσεων IP. Κάτω από το στιγμιότυπο οθόνης δείχνει πώς μπορείτε να ρυθμίσετε την ετικέτα DNS για τη δημόσια διεύθυνση IP. 

![Ετικέτα DNS](./media/active-directory-adfs-in-azure-with-azure-traffic-manager/eastfabstsdnslabel.png)

###<a name="deploying-azure-traffic-manager"></a>Ανάπτυξη Azure Manager κυκλοφορία

Ακολουθήστε τα παρακάτω βήματα για τη δημιουργία προφίλ manager κίνηση. Για περισσότερες πληροφορίες, μπορείτε επίσης να ανατρέξετε στην επιλογή [Διαχείριση προφίλ του Azure κυκλοφορία Manager](../traffic-manager/traffic-manager-manage-profiles.md).

1. **Δημιουργία προφίλ Manager κίνηση:** Δώστε το προφίλ σας κίνηση manager ένα μοναδικό όνομα. Αυτό το όνομα του προφίλ είναι μέρος του ονόματος του DNS και λειτουργεί ως ένα πρόθεμα για την ετικέτα ονόματος τομέα Manager κίνηση. Το όνομα / προστίθεται πρόθεμα. trafficmanager.net για να δημιουργήσετε μια ετικέτα DNS για την κίνηση manager. Το παρακάτω στιγμιότυπο οθόνης εμφανίζει τη Διαχείριση κίνηση πρόθεμα DNS να ορίσετε ως mysts και που προκύπτει ετικέτα DNS θα mysts.trafficmanager.net. 

    ![Δημιουργία προφίλ Manager κίνηση](./media/active-directory-adfs-in-azure-with-azure-traffic-manager/trafficmanager01.png)
 
2. **Κυκλοφορίας μέθοδο δρομολόγησης:** Υπάρχουν τρεις επιλογές δρομολόγησης που είναι διαθέσιμες στη Διαχείριση κίνηση:

    * Προτεραιότητα 
    * Επιδόσεων
    * Σταθμισμένων μέσων όρων
    
    Η **απόδοση** είναι η προτεινόμενη επιλογή για να επιτύχετε ιδιαίτερα αποκρίνεται υποδομή AD FS. Ωστόσο, μπορείτε να επιλέξετε οποιοδήποτε μέθοδο δρομολόγησης καταλληλότερες για τις ανάγκες σας ανάπτυξης. Η λειτουργικότητα AD FS δεν επηρεάζεται από την επιλογή δρομολόγηση. Για περισσότερες πληροφορίες, ανατρέξτε στο θέμα [δρομολόγηση μεθόδους κίνηση Manager κίνηση](../traffic-manager/traffic-manager-routing-methods.md) . Στο δείγμα στιγμιότυπο οθόνης παραπάνω μπορείτε να δείτε τη μέθοδο **επιδόσεων** που επιλέξατε.
   
3.  **Ρύθμιση παραμέτρων τελικά σημεία:** Στη σελίδα Διαχείριση κίνηση, κάντε κλικ στην εντολή τελικά σημεία και επιλέξτε Προσθήκη. Αυτό θα ανοίξει μια σελίδα τελικού σημείου Προσθήκη παρόμοιο με το παρακάτω στιγμιότυπο οθόνης
 
    ![Ρύθμιση παραμέτρων τελικά σημεία](./media/active-directory-adfs-in-azure-with-azure-traffic-manager/eastfsendpoint.png)
 
    Για το διαφορετικό εισροές, ακολουθήστε τα παρακάτω κατευθυντήριας γραμμής:

    **Τύπος:** Επιλέξτε Azure τελικού σημείου όπως θα σας θα που δείχνει μια Azure δημόσια διεύθυνση IP.

    **Όνομα:** Δημιουργήστε ένα όνομα που θέλετε να συσχετίσετε με το τελικό σημείο. Αυτό δεν είναι το όνομα DNS και δεν έχει καμία που φέρει τις εγγραφές DNS.

    **Στόχου τύπος πόρου:** Επιλέξτε δημόσια διεύθυνση IP ως η τιμή σε αυτήν την ιδιότητα. 

    **Προορισμού πόρων:** Αυτό, θα σας δώσει μια επιλογή για να επιλέξετε από τις διαφορετικές ετικέτες DNS που είναι διαθέσιμες στην περιοχή τη συνδρομή σας. Επιλέξτε την ετικέτα DNS για να.

    Προσθήκη τελικό σημείο για κάθε γεωγραφική περιοχή που θέλετε στη Διαχείριση κίνηση Azure να κυκλοφορία δρομολόγηση στο.
    Για περισσότερες πληροφορίες και λεπτομερείς οδηγίες σχετικά με τον τρόπο για να προσθέσετε / ρύθμιση παραμέτρων τελικά σημεία στη Διαχείριση κυκλοφορία, ανατρέξτε στο θέμα για να [προσθέσετε, απενεργοποιήσετε, να ενεργοποιήσετε ή να διαγράψετε τα τελικά σημεία](../traffic-manager/traffic-manager-endpoints.md)
    
4. **Δοκιμή του ρύθμιση παραμέτρων:** Στη σελίδα Διαχείριση κίνηση, κάντε κλικ στην επιλογή ρύθμιση των παραμέτρων του. Στη σελίδα Ρύθμιση παραμέτρων, πρέπει να αλλάξετε τις ρυθμίσεις της οθόνης για τη Διερεύνηση στο HTTP θύρες 80 και σχετική διαδρομή /adfs/probe

    ![Ρύθμιση παραμέτρων δοκιμή του](./media/active-directory-adfs-in-azure-with-azure-traffic-manager/mystsconfig.png) 

    >[AZURE.NOTE] **Βεβαιωθείτε ότι η κατάσταση της τα τελικά σημεία είναι ONLINE μόλις ολοκληρωθεί η ρύθμιση παραμέτρων**. Εάν όλα τα τελικά σημεία σε κατάσταση 'υποβαθμίζονται', Azure κίνηση Manager θα κάνετε μια καλύτερη προσπάθεια για να δρομολογήσετε την κυκλοφορία υπό την προϋπόθεση ότι ότι τα Διαγνωστικά είναι εσφαλμένη και όλα τα τελικά σημεία είναι προσβάσιμος.

5. **Εγγραφές DNS τροποποίηση για τη Διαχείριση κίνηση Azure:** Υπηρεσία Ομοσπονδία πρέπει να είναι μια εγγραφή CNAME για το όνομα DNS Manager κίνηση Azure. Δημιουργήσετε μια εγγραφή CNAME στις δημόσιες εγγραφές DNS, ώστε να οποιουδήποτε ατόμου προσπαθεί να επικοινωνήσει με την υπηρεσία Ομοσπονδία φτάσει πραγματικά στην τη Διαχείριση κυκλοφορία Azure.

    Για παράδειγμα, ώστε να οδηγούν την Ομοσπονδία fs.fabidentity.com υπηρεσίας για τη Διαχείριση κίνηση, θα πρέπει να ενημερώσετε την εγγραφή DNS πόρων για να είναι τα εξής:

    <code>fs.fabidentity.com IN CNAME mysts.trafficmanager.net</code>

##<a name="test-the-routing-and-ad-fs-sign-in"></a>Δοκιμή του δρομολόγηση και AD FS εισόδου   

###<a name="routing-test"></a>Δρομολόγηση δοκιμής

Ένα βασικό έλεγχο για τη δρομολόγηση είναι να δοκιμάσετε κάντε ping το όνομα Ομοσπονδία υπηρεσίας DNS από έναν υπολογιστή σε κάθε γεωγραφική περιοχή. Ανάλογα με τη μέθοδο δρομολόγησης επιλέξει, το τελικό σημείο στην πραγματικότητα κάνει ping θα εμφανιστούν στην εμφάνιση ping. Για παράδειγμα, εάν έχετε επιλέξει τη δρομολόγηση επιδόσεων, στη συνέχεια, το τελικό σημείο κοντά της περιοχής του προγράμματος-πελάτη θα δυνατή η πρόσβαση. Ακολουθεί το στιγμιότυπο της δύο ping από δύο υπολογιστές-πελάτες διαφορετική περιοχή, μία στην περιοχή EastAsia και μία στη δυτική ΗΠΑ. 

![Δρομολόγηση δοκιμής](./media/active-directory-adfs-in-azure-with-azure-traffic-manager/pingtest.png)

###<a name="ad-fs-sign-in-test"></a>AD FS εισόδου δοκιμής

Ο ευκολότερος τρόπος για να ελέγξετε AD FS είναι χρησιμοποιώντας τη σελίδα IdpInitiatedSignon.aspx. Για να έχετε τη δυνατότητα να κάνετε ότι, απαιτείται για να ενεργοποιήσετε το IdpInitiatedSignOn σχετικά με τις ιδιότητες AD FS. Ακολουθήστε τα παρακάτω βήματα για να επαληθεύσετε τις ρυθμίσεις σας AD FS
 
1. Εκτελέστε την παρακάτω cmdlet στο διακομιστή AD FS, χρησιμοποιώντας το PowerShell, για να ορίσετε την επιλογή ενεργοποιημένη. Ορισμός AdfsProperties - EnableIdPInitiatedSignonPage $true
2. Από οποιαδήποτε https:// πρόσβαση εξωτερικών μηχανική<yourfederationservicedns>/adfs/ls/IdpInitiatedSignon.aspx
3. Θα πρέπει να δείτε τη σελίδα AD FS όπως παρακάτω:

    ![Δοκιμή ADFS - πρόκληση ελέγχου ταυτότητας](./media/active-directory-adfs-in-azure-with-azure-traffic-manager/adfstest1.png)

    και επιτυχής είσοδος, αυτό θα σας δώσει ένα μήνυμα επιτυχίας όπως φαίνεται παρακάτω:

    ![Δοκιμή ADFS - επιτυχίας ελέγχου ταυτότητας](./media/active-directory-adfs-in-azure-with-azure-traffic-manager/adfstest2.png)
