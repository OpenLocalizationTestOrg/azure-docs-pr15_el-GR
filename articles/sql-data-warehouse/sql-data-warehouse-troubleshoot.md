# <a name="troubleshooting-azure-sql-data-warehouse"></a>Αντιμετώπιση προβλημάτων αποθήκη δεδομένων του Azure SQL

Αυτό το θέμα παραθέτει ορισμένες από τις πιο συνήθεις ερωτήσεις αντιμετώπισης προβλημάτων που θα σας ακούσουμε οι πελάτες μας.

## <a name="connecting"></a>Σύνδεση

| Το ζήτημα                              | Ανάλυση                                      |
| :----------------------------------| :---------------------------------------------- |
| Απέτυχε η σύνδεση για το χρήστη 'NT ανώνυμη ΣΎΝΔΕΣΗ'. (Microsoft SQL Server, σφάλματος: 18456) | Αυτό το σφάλμα παρουσιάζεται όταν ένας χρήστης AAD προσπαθεί να συνδεθεί με την κύρια βάση δεδομένων, αλλά δεν διαθέτει ένα χρήστη στο υπόδειγμα.  Για να διορθώσετε αυτό το ζήτημα είτε καθορίστε την αποθήκη δεδομένων του SQL που θέλετε να συνδεθείτε με κατά το χρόνο σύνδεσης ή να προσθέσετε το χρήστη με την κύρια βάση δεδομένων.  Ανατρέξτε στο θέμα [Επισκόπηση ασφαλείας][] άρθρου για περισσότερες λεπτομέρειες.|
|Ο διακομιστής κεφαλαίου "MyUserName που ορίζετε" δεν μπορούν να έχουν πρόσβαση στη βάση δεδομένων "υπόδειγμα" στο τρέχον περιβάλλον ασφαλείας. Δεν είναι δυνατό να ανοίξετε χρήστη προεπιλεγμένης βάσης δεδομένων. Απέτυχε η σύνδεση. Απέτυχε η σύνδεση για το χρήστη 'MyUserName ' που ορίζετε. (Microsoft SQL Server, σφάλματος: 916) | Αυτό το σφάλμα παρουσιάζεται όταν ένας χρήστης AAD προσπαθεί να συνδεθεί με την κύρια βάση δεδομένων, αλλά δεν διαθέτει ένα χρήστη στο υπόδειγμα.  Για να διορθώσετε αυτό το ζήτημα είτε καθορίστε την αποθήκη δεδομένων του SQL που θέλετε να συνδεθείτε με κατά το χρόνο σύνδεσης ή να προσθέσετε το χρήστη με την κύρια βάση δεδομένων.  Ανατρέξτε στο θέμα [Επισκόπηση ασφαλείας][] άρθρου για περισσότερες λεπτομέρειες.|
| Σφάλμα CTAIP                        | Αυτό το σφάλμα μπορεί να προκύψει όταν έχει δημιουργηθεί μια σύνδεση στην κύρια βάση δεδομένων του SQL server, αλλά όχι σε βάση δεδομένων SQL αποθήκη δεδομένων.  Εάν αντιμετωπίσετε αυτό το σφάλμα, Ρίξτε μια ματιά στο άρθρο [Επισκόπηση ασφαλείας][] .  Σε αυτό το άρθρο εξηγεί τον τρόπο δημιουργίας δημιουργία σύνδεσης και χρήστη στο υπόδειγμα και, στη συνέχεια, τον τρόπο δημιουργίας ενός χρήστη στη βάση δεδομένων SQL αποθήκη δεδομένων.|
| Αποκλείονται από το τείχος προστασίας                |Βάσεις δεδομένων του SQL Azure προστατεύονται με διακομιστής και βάση δεδομένων επιπέδου τείχη προστασίας για να βεβαιωθείτε ότι μόνο γνωστά διευθύνσεις IP που έχουν πρόσβαση σε μια βάση δεδομένων. Τα τείχη προστασίας είναι ασφαλείς από προεπιλογή, γεγονός που σημαίνει ότι πρέπει να ενεργοποιήσετε ρητά και τη διεύθυνση IP ή περιοχή διευθύνσεων για να μπορέσετε να συνδεθείτε.  Για να ρυθμίσετε τις παραμέτρους του τείχους προστασίας για την πρόσβαση, ακολουθήστε τα βήματα στη [Ρύθμιση παραμέτρων διακομιστή τείχος προστασίας πρόσβασης για το IP υπολογιστή-πελάτη][] στην [Provisioning οδηγίες][].|
| Δεν μπορεί να συνδεθεί με το εργαλείο ή πρόγραμμα οδήγησης | Αποθήκη δεδομένων του SQL συνιστά τη χρήση [SSMS][], [SSDT Visual Studio 2015][]ή [sqlcmd][] ερώτημα για τα δεδομένα σας. Για περισσότερες λεπτομέρειες σχετικά με προγράμματα οδήγησης και τη σύνδεση σε αποθήκη δεδομένων του SQL, ανατρέξτε στο θέμα [προγράμματα οδήγησης για αποθήκη δεδομένων του SQL Azure][] και [σύνδεση με αποθήκη δεδομένων του SQL Azure][] άρθρα.|

|  Το ζήτημα                             | Ανάλυση                                      |
| :----------------------------------| :---------------------------------------------- |
| Αντιμετώπιση προβλημάτων στις επιδόσεις ερωτημάτων  | Εάν προσπαθείτε να αντιμετωπίσετε ένα συγκεκριμένο ερώτημα, ξεκινήστε με [μαθαίνοντας πώς να παρακολουθείτε τα ερωτήματά σας][].|
| Προγράμματα και επιδόσεις ερωτημάτων κακή συχνά είναι ένα αποτέλεσμα που λείπουν στατιστικών στοιχείων   | Τα πιο συνηθισμένα αιτία χαμηλές επιδόσεις είναι έλλειψης στατιστικά στοιχεία σχετικά με τους πίνακές σας.  Δείτε τα [Στατιστικά στοιχεία πίνακα Maintaining] [ Statistics] για λεπτομέρειες σχετικά με τον τρόπο δημιουργίας στατιστικών στοιχείων και γιατί είναι σημαντικό για τις επιδόσεις του.|
| Χαμηλή ταυτόχρονης εκτέλεσης / στην ουρά ερωτημάτων   | Κατανόηση των [Διαχείριση φόρτου εργασίας][] είναι σημαντικό για να κατανοήσετε τον τρόπο για να εξισορροπήσετε εκχώρηση μνήμης με ταυτόχρονης εκτέλεσης.|
| Πώς μπορείτε να υλοποιήσετε βέλτιστες πρακτικές    | Τοποθετήστε το καλύτερο για να ξεκινήσετε για να μάθετε τρόπους για να βελτιώσετε την απόδοση του ερωτήματος είναι άρθρο [βέλτιστες πρακτικές αποθήκη δεδομένων του SQL][] .|
| Πώς μπορείτε να βελτιώσετε την απόδοση με κλιμάκωση  | Μερικές φορές η λύση για τη βελτίωση της απόδοσης είναι απλώς να προσθέσετε περισσότερα υπολογιστική ισχύ στα ερωτήματά σας, [αλλάζοντας την κλίμακα σας αποθήκη δεδομένων του SQL][].|
| Επιδόσεις ερωτημάτων κακή λόγω ευρετηρίου κακή ποιότητα | Ορισμένες φορές ερωτήματα να επιβράδυνση λόγω [columnstore κακή ποιότητα ευρετηρίου][].  Ανατρέξτε σε αυτό το άρθρο για περισσότερες πληροφορίες και πώς να [εκ νέου δημιουργία ευρετηρίων για να βελτιώσω την ποιότητα του τμήματος][].|

| Βοήθεια για τη Διαχείριση πινάκων          | Δείτε την [Επισκόπηση πίνακα] [ Overview] το άρθρο για βοήθεια σχετικά με τη Διαχείριση τους πίνακές σας.  Σε αυτό το άρθρο περιλαμβάνει επίσης συνδέσεις σε πιο λεπτομερή θέματα όπως [τύπους δεδομένων πίνακα][Data types], [διανομή πίνακα][Distribute], [δημιουργίας ευρετηρίου για έναν πίνακα][Index], [διαμερισμάτων πίνακα][Partition], [Στατιστικά στοιχεία πίνακα Maintaining] [ Statistics] και [προσωρινό πίνακες][Temporary].|

## <a name="polybase"></a>Polybase

|  Το ζήτημα                             | Ανάλυση                                      |
| :----------------------------------| :---------------------------------------------- |
| Σφάλμα UTF-8                        |  Προς το παρόν PolyBase υποστηρίζει μόνο κατά τη φόρτωση των αρχείων δεδομένων που έχουν κωδικοποίηση UTF-8.  Για οδηγίες σχετικά με τον τρόπο για να επιλύσετε αυτόν τον περιορισμό, ανατρέξτε στο θέμα [εργασία γύρω από την απαίτηση PolyBase UTF-8][] .|
| Φόρτωση αποτυγχάνει λόγω μεγάλο γραμμών   | Αυτήν τη στιγμή δεν είναι διαθέσιμη για Polybase μεγάλη γραμμή υποστήριξης.  Αυτό σημαίνει ότι εάν ο πίνακας περιέχει VARCHAR(MAX), NVARCHAR(MAX) ή VARBINARY(MAX), εξωτερικοί πίνακες δεν μπορούν να χρησιμοποιηθούν για τη φόρτωση των δεδομένων σας.  Φορτία για μεγάλο γραμμών είναι αυτήν τη στιγμή υποστηρίζεται μόνο μέσω Azure εργοστασίου δεδομένων (με BCP), Azure ροή ανάλυση, SSIS, BCP ή την κλάση .NET SQLBulkCopy. PolyBase υποστήριξη για μεγάλα γραμμές θα προστεθεί σε μελλοντική έκδοση.|
| αποτυγχάνει BCP φόρτωσης του πίνακα με τον τύπο δεδομένων MAX | Υπάρχει ένα γνωστό θέμα που απαιτεί την VARCHAR(MAX), NVARCHAR(MAX) ή VARBINARY(MAX) τοποθέτηση στο τέλος του πίνακα σε ορισμένα σενάρια.  Δοκιμάστε να μετακινήσετε τις στήλες MAX μέχρι το τέλος του πίνακα.|

## <a name="differences-from-sql-database"></a>Διαφορές από βάση δεδομένων SQL

|  Το ζήτημα                             | Ανάλυση                                      |
| :----------------------------------| :---------------------------------------------- |
| Μη υποστηριζόμενες δυνατότητες βάσης δεδομένων SQL  | Ανατρέξτε στο θέμα [μη υποστηριζόμενες δυνατότητες πίνακα][].|
| Μη υποστηριζόμενους τύπους δεδομένων βάσης δεδομένων SQL  | Ανατρέξτε στο θέμα [μη υποστηριζόμενους τύπους δεδομένων][].|
| ΔΙΑΓΡΑΦΉ και τους περιορισμούς ΕΝΗΜΈΡΩΣΗΣ      | Ανατρέξτε στο θέμα [λύσεις ΕΝΗΜΈΡΩΣΗ][], [ΔΙΑΓΡΑΦΉ λύσεις][] και [Χρήση CTAS για να επιλύσετε δεν υποστηρίζεται σύνταξη ΕΝΗΜΈΡΩΣΗ και ΔΙΑΓΡΑΦΉ][].  |
| Εντολή ΣΥΓΧΏΝΕΥΣΗ δεν υποστηρίζεται   | Ανατρέξτε στο θέμα [ΣΥΓΧΏΝΕΥΣΗ λύσεις][].|
| Περιορισμοί αποθηκευμένη διαδικασία       | Ανατρέξτε στο θέμα [περιορισμοί αποθηκευμένη διαδικασία][] για να κατανοήσετε ορισμένοι από τους περιορισμούς των αποθηκευμένων διαδικασιών.|
| UDF δεν υποστηρίζουν προτάσεις SELECT | Αυτή είναι μια τρέχουσα τον περιορισμό μας UDF.  Ανατρέξτε στο θέμα [ΔΗΜΙΟΥΡΓΊΑ ΣΥΝΆΡΤΗΣΗ][] για τη σύνταξη υποστηρίζουμε.   |
