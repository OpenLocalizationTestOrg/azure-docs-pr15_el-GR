# <a name="sql-data-warehouse-capacity-limits"></a>Όρια χωρητικότητα αποθήκη δεδομένων του SQL

Στους παρακάτω πίνακες περιέχουν τις μέγιστες τιμές που επιτρέπονται για διάφορα στοιχεία του αποθήκη δεδομένων του SQL Azure.


## <a name="workload-management"></a>Διαχείριση φόρτου εργασίας

| Κατηγορία            | Περιγραφή                                       | Μέγιστη            |
| :------------------ | :------------------------------------------------ | :----------------- |
| [Μονάδες αποθήκη δεδομένων (DWU)][]| Max DWU για μια μεμονωμένη αποθήκη δεδομένων SQL | 6000               |
| [Μονάδες αποθήκη δεδομένων (DWU)][]| Max DWU για μια μεμονωμένη SQL server         | 6000 από προεπιλογή<br/><br/> Από προεπιλογή, κάθε SQL server (π.χ., myserver.database.windows.net) διαθέτει ένα όριο DTU των 45,000 που επιτρέπει έως και 6000 DWU. Αυτό το όριο είναι απλώς ένα όριο ασφάλεια. Μπορείτε να αυξήσετε το όριο, [δημιουργώντας ένα δελτίο υποστήριξης][] και επιλογή *ορίου* ως τον τύπο αίτησης.  Για να υπολογίσετε το DTU ανάγκες, πρέπει να πολλαπλασιάσετε το 7.5 από το σύνολο DWU χρειάζεται. Μπορείτε να προβάλετε την τρέχουσα κατανάλωση DTU από το blade SQL server στην πύλη. Καταμέτρηση τόσο παύσης και κατάργησης παύσης βάσεις δεδομένων προς το όριο DTU. |
| Σύνδεση με βάση δεδομένων | Ταυτόχρονες ανοιχτών περιόδων λειτουργίας                          | 1024<br/><br/>Υποστηρίζουμε έως 1024 ενεργές συνδέσεις, κάθε μία από τις οποίες μπορούν να υποβάλετε αιτήσεις σε μια βάση δεδομένων SQL Data Warehouse την ίδια στιγμή. Σημειώστε ότι δεν υπάρχουν περιορισμοί στον αριθμό των ερωτημάτων που πραγματικά μπορούν να εκτελεστούν ταυτόχρονα. Όταν γίνει υπέρβαση του ορίου ταυτόχρονης εκτέλεσης, την αίτηση μεταβαίνει σε μια εσωτερική ουρά όπου αναμένει για επεξεργασία.|
| Σύνδεση με βάση δεδομένων | Μέγιστης μνήμης για έτοιμων προτάσεων            | 20 MB              |
| [Διαχείριση φόρτου εργασίας][] | Μέγιστος αριθμός ταυτόχρονες ερωτημάτων                    | 32<br/><br/> Από προεπιλογή, αποθήκη δεδομένων του SQL μπορεί να εκτελέσει έως 32 ταυτόχρονες ερωτήματα και ουρές υπόλοιπη ερωτήματα.<br/><br/>Το επίπεδο ταυτόχρονης εκτέλεσης μπορεί να μειωθεί όταν οι χρήστες έχουν εκχωρηθεί σε κλάση πόρων υψηλότερη ή όταν αποθήκη δεδομένων του SQL έχει ρυθμιστεί με χαμηλή DWU. Ορισμένα ερωτήματα, όπως ερωτήματα DMV, πάντα επιτρέπεται η εκτέλεση.|
| [Tempdb][]          | Μέγιστο μέγεθος του Tempdb                                | 399 GB ανά DW100. Επομένως, στο DWU1000 Tempdb μέγεθος 3,99 TB |


## <a name="database-objects"></a>Αντικείμενα της βάσης δεδομένων

| Κατηγορία          | Περιγραφή                                  | Μέγιστη            |
| :---------------- | :------------------------------------------- | :----------------- |
| Βάση δεδομένων          | Μέγιστο μέγεθος                                     | 240 TB συμπιέσει στο δίσκο<br/><br/>Αυτός ο χώρος είναι ανεξάρτητο από χώρο tempdb ή το αρχείο καταγραφής και, επομένως, αυτός ο χώρος είναι αφοσιωμένη στο να μόνιμο πίνακες.  Συμπίεση ομαδοποιημένων columnstore υπολογίζεται σε 5 X.  Αυτό συμπίεσης επιτρέπει τη βάση δεδομένων θα αυξηθεί στο περίπου 1 PB όταν όλοι οι πίνακες είναι ομαδοποιημένη columnstore (ο προεπιλεγμένος τύπος πίνακα).|
| Πίνακας             | Μέγιστο μέγεθος                                     | 60 TB συμπιέσει στο δίσκο   |
| Πίνακας             | Πίνακες ανά βάση δεδομένων                          | 2 δισεκατομμύρια          |
| Πίνακας             | Στηλών ανά πίνακα                            | 1024 στηλών       |
| Πίνακας             | Byte ανά στήλη                             | Εξαρτάται από τη στήλη [Τύπος δεδομένων][].  Το όριο είναι 8000 για τύπους δεδομένων char, 4000 για nvarchar ή 2 GB για τους τύπους δεδομένων MAX.|
| Πίνακας             | Byte ανά γραμμή, καθορισμένο μέγεθος                  | 8060 byte<br/><br/>Το πλήθος των byte ανά γραμμή υπολογίζεται με τον ίδιο τρόπο, καθώς πρόκειται για τον SQL Server με συμπίεση σελίδας. Όπως SQL Server, αποθήκη δεδομένων του SQL υποστηρίζει αποθήκευσης υπερχείλισης γραμμής που σας επιτρέπει **μεταβλητού μήκους στήλες** για να είναι δυνατή η προώθηση απενεργοποίηση γραμμής. Όταν μεταβλητού μήκους γραμμών προωθούνται απενεργοποίηση γραμμής, μόνο 24 byte ριζικό αποθηκεύεται στην κύρια εγγραφή. Για περισσότερες πληροφορίες, ανατρέξτε στο άρθρο MSDN [υπερχείλισης γραμμής δεδομένων άνω των 8 KB][] .|
| Πίνακας             | Τα διαμερίσματα ανά πίνακα                    | 15.000<br/><br/>Για υψηλές επιδόσεις, συνιστάται να ελαχιστοποιήσετε τον αριθμό των διαμερισμάτων που χρειάζεστε κατά εξακολουθεί να υποστηρίζει ανάγκες της επιχείρησής σας. Καθώς εξελίσσεται ο αριθμός των διαμερισμάτων, η εργασία για λειτουργίες γλώσσα ορισμού δεδομένων (DDL) και γλώσσα χειρισμού δεδομένων (DML) μεγαλώνει και έχει ως αποτέλεσμα πιο αργές επιδόσεις.|
| Πίνακας             | Αριθμός χαρακτήρων ανά τιμή όριο διαμερίσματα.| 4000 |
| Ευρετήριο             | Ομαδοποιημένη μη ευρετηρίων ανά πίνακα.        | 999<br/><br/>Ισχύει για πίνακες rowstore μόνο.|
| Ευρετήριο             | Ομαδοποιημένη ευρετηρίων ανά πίνακα.            | 1<br><br/>Ισχύει για πίνακες rowstore και columnstore.|
| Ευρετήριο             | Μέγεθος κλειδιού ευρετηρίου.                          | 900 byte.<br/><br/>Ισχύει για τα ευρετήρια rowstore μόνο.<br/><br/>Τα ευρετήρια στις στήλες varchar με μέγιστο μέγεθος των περισσότερους από 900 byte μπορούν να δημιουργηθούν εάν τα υπάρχοντα δεδομένα στις στήλες δεν υπερβαίνει τα 900 byte όταν δημιουργείται το ευρετήριο. Ωστόσο, ΕΙΣΑΓΆΓΕΤΕ αργότερα ή ΕΝΗΜΈΡΩΣΗ ενέργειες στις στήλες που προκαλούν το συνολικό μέγεθος για να υπερβαίνει τα 900 byte θα αποτύχει.|
| Ευρετήριο             | Βασικές στήλες ανά ευρετήριο.                   | 16<br/><br/>Ισχύει για τα ευρετήρια rowstore μόνο. Ευρετήρια ομαδοποιημένων columnstore περιλαμβάνουν όλες τις στήλες.|
| Στατιστικά στοιχεία        | Οι τιμές της στήλης συνολικό μέγεθος.      | 900 byte.         |
| Στατιστικά στοιχεία        | Στήλες ανά αντικείμενο στατιστικά στοιχεία.           | 32                 |
| Στατιστικά στοιχεία        | Στατιστικά στοιχεία που δημιουργήθηκε σε στηλών ανά πίνακα. | 30.000            |
| Αποθηκευμένες διαδικασίες | Μέγιστα επίπεδα ένθεσης.               | 8                 |
| Προβολή              | Στήλες ανά προβολή                         | 1.024             |


## <a name="loads"></a>Φορτία

| Κατηγορία          | Περιγραφή                                  | Μέγιστη            |
| :---------------- | :------------------------------------------- | :----------------- |
| Polybase φορτία    | Byte ανά γραμμή                                | 32.768<br/><br/>Polybase φορτία περιορίζονται για τη φόρτωση γραμμές και τις δύο μικρότερες από 32K και δεν είναι δυνατή η φόρτωση VARCHR(MAX), NVARCHAR(MAX) ή VARBINARY(MAX).  Ενώ αυτό το όριο υπάρχει σήμερα, θα καταργηθεί αρκετά σύντομα.<br/><br/>


## <a name="queries"></a>Ερωτήματα

| Κατηγορία          | Περιγραφή                                  | Μέγιστη            |
| :---------------- | :------------------------------------------- | :----------------- |
| Ερώτημα             | Σε ουρά ερωτήματα σε πίνακες χρήστη.               | 1000               |
| Ερώτημα             | Ταυτόχρονες ερωτήματα σχετικά με τις προβολές συστήματος.          | 100                |
| Ερώτημα             | Σε ουρά ερωτήματα σχετικά με τις προβολές συστήματος               | 1000               |
| Ερώτημα             | Μέγιστο παραμέτρους                           | 2098               |
| Μαζική             | Μέγιστο μέγεθος                                 | 65, 536 * 4096        |
| ΕΠΙΛΈΞΤΕ αποτελεσμάτων    | Στηλών ανά γραμμή                              | 4096<br/><br/>Μπορείτε να έχετε ποτέ περισσότερα από 4096 στηλών ανά γραμμή στο αποτέλεσμα ΕΠΙΛΟΓΉΣ. Δεν υπάρχει εγγύηση ότι μπορείτε να έχετε πάντα 4096. Εάν το πρόγραμμα στο ερώτημα απαιτεί ένα προσωρινό πίνακα, ενδέχεται να ισχύουν τα 1024 στηλών ανά πίνακα μέγιστο.|
| ΕΠΙΛΈΞΤΕ            | Ένθετου δευτερεύοντα ερωτήματα                            | 32<br/><br/>Δεν μπορείτε να έχετε περισσότερους από 32 ένθετου δευτερεύοντα ερωτήματα σε μια πρόταση SELECT. Δεν υπάρχει εγγύηση ότι μπορείτε να έχετε πάντα 32. Για παράδειγμα, ένα ΣΎΝΔΕΣΜΟ μπορεί να εισαγάγει ένα δευτερεύον ερώτημα στο σχεδιασμό του ερωτήματος. Τον αριθμό των δευτερεύοντα ερωτήματα μπορεί επίσης να περιορίζεται από τη διαθέσιμη μνήμη.|
| ΕΠΙΛΈΞΤΕ            | Στήλες ανά ΣΥΝΔΈΣΜΟΥ                             | 1024 στηλών<br/><br/>Μπορείτε να έχετε ποτέ μεγαλύτερο από 1024 στηλών του συνδέσμου. Δεν υπάρχει εγγύηση ότι μπορείτε να έχετε πάντα 1024. Εάν το πρόγραμμα ΣΎΝΔΕΣΜΟΣ απαιτεί ένα προσωρινό πίνακα με περισσότερες στήλες παρά το αποτέλεσμα του ΣΥΝΔΈΣΜΟΥ, το όριο των 1024 ισχύει για τον προσωρινό πίνακα. |
| ΕΠΙΛΈΞΤΕ            | Byte ανά στήλες GROUP BY.                  | 8060<br/><br/>Οι στήλες στον όρο GROUP BY μπορεί να έχει έως 8060 byte.|
| ΕΠΙΛΈΞΤΕ            | Byte ανά ΣΕΙΡΆ κατά στήλες                   | 8060 byte.<br/><br/>Οι στήλες ο όρος ORDER BY μπορεί να έχει έως 8060 byte.|
| Αναγνωριστικά και σταθερών ανά δήλωση | Αριθμός αναφοράς αναγνωριστικά και σταθερές. | 65.535<br/><br/>Αποθήκη δεδομένων του SQL περιορίζει τον αριθμό των αναγνωριστικών και σταθερές, οι οποίες μπορεί να περιέχονται σε μια μεμονωμένη παράσταση ενός ερωτήματος. Αυτό το όριο είναι 65,535. Υπερβαίνει αυτό αριθμού αποτέλεσμα σφάλμα SQL Server 8632. Για περισσότερες πληροφορίες, ανατρέξτε στο θέμα [εσωτερικό σφάλμα: μια παράσταση υπηρεσίες όριο][].|
