# <a name="set-up-an-azure-virtual-machine-as-an-ipython-notebook-server-for-advanced-analytics"></a>Ρυθμίσετε μια εικονική μηχανή Azure ως ένα σημειωματάριο IPython διακομιστή για προηγμένη ανάλυση

Αυτό το θέμα δείχνει πώς μπορείτε να προμηθεύσουν και να ρυθμίσετε μια εικονική μηχανή Azure για προηγμένη ανάλυση που μπορούν να χρησιμοποιηθούν ως μέρος ενός περιβάλλοντος science δεδομένων. Η εικονική μηχανή Windows έχει ρυθμιστεί με εργαλεία όπως όπως το Σημειωματάριο IPython Azure Εξερεύνηση χώρου αποθήκευσης, AzCopy, καθώς και άλλα βοηθητικά προγράμματα που είναι χρήσιμες για προηγμένη ανάλυση έργα υποστήριξης. Azure Εξερεύνηση χώρου αποθήκευσης και AzCopy, για παράδειγμα, παρέχει εύκολους τρόπους για την αποστολή δεδομένων στο χώρο αποθήκευσης αντικειμένων blob του Azure από τον τοπικό σας υπολογιστή ή για να κάνετε λήψη της στον τοπικό υπολογιστή σας από το χώρο αποθήκευσης αντικειμένων blob.

## <a name="create-vm"></a>Βήμα 1: Δημιουργία μιας γενικής χρήσης Azure εικονική μηχανή

Εάν έχετε ήδη μια εικονική μηχανή Azure και απλώς θέλετε να ρυθμίσετε ένα διακομιστή IPython σημειωματαρίου σε αυτό, μπορείτε να παραλείψετε αυτό το βήμα και να συνεχίσετε να [βήμα 2: Προσθήκη ένα τελικό σημείο για τα σημειωματάρια IPython σε μια υπάρχουσα εικονική μηχανή](#add-endpoint).

Πριν ξεκινήσετε τη διαδικασία για τη δημιουργία μια εικονική μηχανή στο Azure, πρέπει να προσδιορίσετε το μέγεθος του υπολογιστή που είναι απαραίτητη για την επεξεργασία των δεδομένων για το έργο. Μικρότερο μηχανές έχουν λιγότερη μνήμη και λιγότερες πυρήνων CPU από μεγαλύτερα μηχανήματα, αλλά είναι επίσης λιγότερο ακριβό. Για μια λίστα των τύπων υπολογιστή και τιμές, ανατρέξτε στη σελίδα <a href="http://azure.microsoft.com/pricing/details/virtual-machines/" target="_blank">Τιμολόγηση εικονικές μηχανές</a>

1. Συνδεθείτε στην <a href="https://manage.windowsazure.com" target="_blank">Πύλη κλασική Azure</a>και κάντε κλικ στην επιλογή " **Δημιουργία** " στην κάτω αριστερή γωνία. Θα εμφανιστεί ένα παράθυρο. Επιλέξτε **τον ΥΠΟΛΟΓΙΣΜΌ** -> **ΕΙΚΟΝΙΚΉ ΜΗΧΑΝΉ** -> **Από τη ΣΥΛΛΟΓΉ**.

    ![Δημιουργία χώρου εργασίας][24]

2. Επιλέξτε μία από τις ακόλουθες εικόνες:

    * Κέντρο δεδομένων του Windows Server 2012 R2
    * Εμπειρία Essentials διακομιστή των Windows (Windows Server 2012 R2)

    Στη συνέχεια, κάντε κλικ στο βέλος που δείχνει δεξιά στην κάτω δεξιά γωνία για να μεταβείτε στην επόμενη σελίδα ρύθμισης παραμέτρων.

    ![Δημιουργία χώρου εργασίας][25]

3. Πληκτρολογήστε ένα όνομα για την εικονική μηχανή που θέλετε να δημιουργήσετε, να επιλέξετε το μέγεθος του υπολογιστή (προεπιλεγμένη: A3) με βάση το μέγεθος των δεδομένων υπολογιστή πρόκειται να διαδικασία και πόσο ισχυρή θέλετε τον υπολογιστή να (μέγεθος μνήμης και τον αριθμό των πυρήνων υπολογισμού), πληκτρολογήστε ένα όνομα χρήστη και τον κωδικό πρόσβασης για τον υπολογιστή. Στη συνέχεια, κάντε κλικ στο βέλος που δείχνει προς τα δεξιά για να μεταβείτε στην επόμενη σελίδα ρύθμισης παραμέτρων.

    ![Δημιουργία χώρου εργασίας][26]

4. Επιλέξτε την **ΠΕΡΙΟΧΉ/ΣΥΝΆΦΕΙΑΣ/ΕΙΚΟΝΙΚΉ ΟΜΆΔΑ ΔΙΚΤΎΟΥ** που περιέχει το **ΛΟΓΑΡΙΑΣΜΌ χώρου ΑΠΟΘΉΚΕΥΣΗΣ** που σχεδιάζετε να χρησιμοποιήσετε για αυτόν τον υπολογιστή εικονικές και, στη συνέχεια, επιλέξτε το λογαριασμό χώρου αποθήκευσης. Προσθέστε ένα τελικό σημείο στο κάτω μέρος του πεδίου **τελικά ΣΗΜΕΊΑ** , πληκτρολογώντας το όνομα του τελικού σημείου ("IPython" εδώ). Μπορείτε να επιλέξετε οποιαδήποτε συμβολοσειρά ως το **ΌΝΟΜΑ** του τελικού σημείου και οποιοσδήποτε ακέραιος αριθμός από 0 έως 65536 που είναι **διαθέσιμα** ως **ΔΗΜΌΣΙΑ ΘΎΡΑ**. Η **ΙΔΙΩΤΙΚΉ ΘΎΡΑ** πρέπει να είναι **9999**. Οι χρήστες θα πρέπει να **αποφύγετε** τη χρήση δημόσιων θύρες που έχουν ήδη εκχωρηθεί για τις υπηρεσίες του internet. <a href="http://www.chebucto.ns.ca/~rakerman/port-table.html" target="_blank">Θύρες για τις υπηρεσίες του Internet</a> παρέχει μια λίστα των θυρών που έχουν εκχωρηθεί και θα πρέπει να αποφεύγονται.

    ![Δημιουργία χώρου εργασίας][27]

5. Κάντε κλικ στο σημάδι ελέγχου για να ξεκινήσετε την εικονική μηχανή προμήθεια διαδικασία.

    ![Δημιουργία χώρου εργασίας][28]


Ενδέχεται να χρειαστούν 15 25 λεπτά για να ολοκληρωθεί η εικονική μηχανή προμήθεια διαδικασία. Αφού δημιουργηθεί η εικονική μηχανή, θα πρέπει να εμφανίζουν την κατάσταση της αυτόν τον υπολογιστή με την **Εκτέλεση**.

![Δημιουργία χώρου εργασίας][29]

## <a name="add-endpoint"></a>Βήμα 2: Προσθήκη ένα τελικό σημείο για τα σημειωματάρια IPython σε μια υπάρχουσα εικονική μηχανή

Εάν έχετε δημιουργήσει την εικονική μηχανή, ακολουθώντας τις οδηγίες στο βήμα 1, στη συνέχεια, το τελικό σημείο για το Σημειωματάριο IPython έχει ήδη προστεθεί και αυτό το βήμα μπορεί να πραγματοποιηθεί παράβλεψη.

Εάν υπάρχει ήδη η εικονική μηχανή και πρέπει να προσθέσετε ένα τελικό σημείο για Σημειωματάριο IPython που πρόκειται να εγκαταστήσετε στο βήμα 3 παρακάτω, πρώτο αρχείο καταγραφής σε κλασική πύλη Azure, επιλέξτε την εικονική μηχανή και προσθέστε το τελικό σημείο για διακομιστή IPython σημειωματαρίου. Η παρακάτω εικόνα περιέχει ένα στιγμιότυπο οθόνης από την πύλη μετά το τελικό σημείο για το Σημειωματάριο IPython έχει προστεθεί σε μια εικονική μηχανή των Windows.

![Δημιουργία χώρου εργασίας][17]

## <a name="run-commands"></a>Βήμα 3: Εγκαταστήστε το Σημειωματάριο IPython και άλλα εργαλεία υποστήριξης

Αφού δημιουργηθεί η εικονική μηχανή, χρησιμοποιήστε το πρωτόκολλο απομακρυσμένης επιφάνειας εργασίας (RDP) για να συνδεθείτε με το Windows εικονική μηχανή. Για οδηγίες, ανατρέξτε στο θέμα [πώς μπορείτε να συνδεθείτε στο μια εικονική μηχανή εκτελεί Windows Server](../virtual-machines/virtual-machines-windows-classic-connect-logon.md). Ανοίξτε τη **γραμμή εντολών** (**μην το παράθυρο εντολών του Powershell**) με δικαιώματα **διαχειριστή** και εκτελέστε την ακόλουθη εντολή.

    set script='https://raw.githubusercontent.com/Azure/Azure-MachineLearning-DataScience/master/Misc/MachineSetup/Azure_VM_Setup_Windows.ps1'

    @powershell -NoProfile -ExecutionPolicy unrestricted -Command "iex ((new-object net.webclient).DownloadString(%script%))"

Όταν ολοκληρωθεί η εγκατάσταση, το διακομιστή IPython Σημειωματάριο ξεκινά αυτόματα σε το *C:\\χρήστες\\\<όνομα χρήστη\>\\έγγραφα\\IPython σημειωματάρια* καταλόγου.

Όταν σας ζητηθεί, πληκτρολογήστε έναν κωδικό πρόσβασης για το Σημειωματάριο IPython και τον κωδικό πρόσβασης του διαχειριστή του υπολογιστή. Αυτή η δυνατότητα επιτρέπει το Σημειωματάριο IPython για την εκτέλεση ως υπηρεσία στον υπολογιστή.

Εδώ, το *& #60, είναι ο αριθμός θύρας δημόσια >* πρέπει να είναι ο αριθμός θύρας που καθορίσατε κατά προστέθηκε το τελικό σημείο IPython σημειωματαρίου.

Το *& #60; όνομα DNS εικονική μηχανή >* μπορείτε να βρείτε στο κλασική πύλη του Azure. Μετά την καταγραφή στο κλασική πύλη, κάντε κλικ στην επιλογή **ΕΙΚΟΝΙΚΈΣ ΜΗΧΑΝΈΣ**, επιλέξτε τον υπολογιστή που δημιουργήσατε και, στη συνέχεια, επιλέξτε **τον πίνακα ΕΡΓΑΛΕΊΩΝ**, το όνομα DNS θα εμφανίζεται ως εξής:

Στη συνέχεια, εισαγάγετε τον κωδικό πρόσβασης που καθορίσατε προηγουμένως για να αποκτήσετε πρόσβαση στα σημειωματάρια IPython.

## <a name="upload"></a>Βήμα 5: Αποστολή ενός υπάρχοντος σημειωματαρίου IPython από ένα τοπικό υπολογιστή με το διακομιστή IPython σημειωματαρίου

Σημειωματάρια IPython παρέχουν έναν εύκολο τρόπο για τους χρήστες για την αποστολή ενός υπάρχοντος σημειωματαρίου IPython στους τοπικούς τους υπολογιστές στο διακομιστή IPython σημειωματαρίου σε τις εικονικές μηχανές. Αφού συνδέονται οι χρήστες στο Σημειωματάριο IPython σε πρόγραμμα περιήγησης web, κάντε κλικ στον **κατάλογο** που θα αποσταλεί το Σημειωματάριο IPython. Στη συνέχεια, επιλέξτε ένα αρχείο σημειωματαρίου IPython .ipynb για την αποστολή από τον τοπικό υπολογιστή στην **Εξερεύνηση αρχείων**, και σύρετε και αποθέστε το στον κατάλογο IPython σημειωματαρίου στο πρόγραμμα περιήγησης web. Κάντε κλικ στο κουμπί **Αποστολή** , για να αποστείλετε το αρχείο .ipynb στο διακομιστή IPython σημειωματαρίου. Οι άλλοι χρήστες, στη συνέχεια, να ξεκινήσετε να χρησιμοποιείτε με από τα προγράμματα περιήγησης web.

##<a name="shutdown"></a>Τερματισμός και καταργήστε την εκχώρηση εικονική μηχανή όταν δεν είναι σε χρήση

Azure εικονικές μηχανές είναι σε τιμές ως **πληρωμή μόνο για το τι μπορείτε να χρησιμοποιήσετε**. Για να εξασφαλίσετε ότι θα είναι δεν που χρεώνονται όταν δεν χρησιμοποιείτε την εικονική μηχανή σας, θα πρέπει να είναι σε κατάσταση **Διακοπή (Deallocated)** όταν δεν είναι σε χρήση.

> [AZURE.NOTE] Εάν τερματίζεται η εικονική μηχανή από μέσα του Εικονική (χρησιμοποιώντας τις επιλογές ενέργειας Windows), η Εικονική έχει διακοπεί αλλά παραμένει που έχει εκχωρηθεί. Για να βεβαιωθείτε ότι δεν θα συνεχίσετε να χρεωθεί, διακόψτε πάντα εικονικές μηχανές από την [Πύλη κλασική Azure](http://manage.windowsazure.com/). Μπορείτε επίσης να διακόψετε την εικονική Μηχανή μέσω του Powershell, καλώντας **ShutdownRoleOperation** με "PostShutdownAction" ισούται με "StoppedDeallocated".

Τερματισμός και εκχώρηση η εικονική μηχανή:

1. Συνδεθείτε [Πύλη κλασική Azure](http://manage.windowsazure.com/) χρησιμοποιώντας το λογαριασμό σας.  

2. Επιλέξτε **ΕΙΚΟΝΙΚΈΣ ΜΗΧΑΝΈΣ** από την αριστερή γραμμή περιήγησης.

3. Στη λίστα των εικονικές μηχανές, κάντε κλικ στο όνομα του υπολογιστή σας εικονικές, στη συνέχεια, μεταβείτε στη σελίδα **πίνακα ΕΡΓΑΛΕΊΩΝ** .

4. Στο κάτω μέρος της σελίδας, κάντε κλικ στο κουμπί **ΤΕΡΜΑΤΙΣΜΟΎ**.

![Εικονική τερματισμού][15]

Η εικονική μηχανή θα κατάργηση εκχώρησης αλλά δεν θα διαγραφούν. Θα μπορεί να επανεκκινήσετε την εικονική μηχανή σας οποιαδήποτε στιγμή από την πύλη κλασική Azure.

## <a name="your-azure-vm-is-ready-to-use-whats-next"></a>Εικονική σας Azure είναι έτοιμη για χρήση: τι ακολουθεί;

Την εικονική μηχανή σας είναι τώρα έτοιμο για χρήση σε σας ασκήσεις science δεδομένων. Η εικονική μηχανή επίσης είναι έτοιμη για χρήση ως ένα σημειωματάριο IPython διακομιστή για την Εξερεύνηση και την επεξεργασία των δεδομένων, καθώς και άλλες εργασίες σε συνδυασμό με το Azure μηχανικής εκμάθησης και η διαδικασία Science δεδομένων ομάδας.

Τα επόμενα βήματα της διαδικασίας Science δεδομένων ομάδας έχουν αντιστοιχιστεί στη [Διαδρομή εκμάθησης](https://azure.microsoft.com/documentation/learning-paths/cortana-analytics-process/) και ενδέχεται να περιλαμβάνει τα βήματα που μεταφορά δεδομένων σε HDInsight, επεξεργασία και εκεί το δείγμα προετοιμασία εκμάθησης από τα δεδομένα με το Azure μηχανικής εκμάθησης.
