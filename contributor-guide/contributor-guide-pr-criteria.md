# <a name="quality-criteria-for-pull-request-review"></a>Κριτήρια ποιότητας για την αίτηση ελκυστική αναθεώρηση

Αυτά τα κριτήρια προορίζονται για συντάκτες που δημιουργούν και συντηρούν τεχνικά άρθρα και ελκυστική αίτηση αναθεωρητές που αναθεώρηση περιεχομένου ελκυστική αιτήσεις. Εάν την αίτησή σας ελκυστική δεν είναι κατάλληλα για [Αυτόματη συγχώνευση](contributor-guide-pull-request-etiquette.md#in-a-hurry-submit-prs-that-can-be-accepted-automatically), αναθεωρηθούν από έναν αναθεωρητή αίτηση human ελκυστική. Οι αναθεωρητές αίτηση ελκυστική αναθεώρηση γενικά μόνο τι είναι το νέο ή τροποποιηθεί. Οι αναθεωρητές αίτηση ελκυστική αξιολογήσετε τις αλλαγές σε ένα αίτημα ελκυστική σύμφωνα με το αποκλεισμού και μη αποκλεισμός ποιότητας αναθεώρηση στοιχείων που αναφέρονται σε αυτό το άρθρο.

## <a name="blocking-content-quality-items"></a>Αποκλεισμός στοιχεία ποιότητας περιεχομένου

Τις ενημερώσεις στην πρόσκληση σε ελκυστική πρέπει να συμμορφώνονται με τα ακόλουθα κριτήρια για συγχώνευση. Οι αναθεωρητές αίτηση ελκυστική παροχή σχολίων στις ελκυστική αίτηση σχόλια για αυτά τα στοιχεία και πληκτρολογήστε `#hold-off` στην πρόσκληση σε ελκυστική για να επιστρέψετε σε εσάς (ο συντάκτης) με σχόλια.

| Κατηγορία | Ποιότητα αναθεώρηση στοιχείου |
|----------|---------------------|
|Προαπαιτούμενα στοιχεία| Το "είστε έτοιμοι να-συγχώνευσης" και οι ετικέτες "επικύρωση ολοκληρώθηκε με επιτυχία" έχουν εκχωρηθεί σε το Προβλ.Προηγ.περ|
|Προαπαιτούμενα στοιχεία| Η αίτηση ελκυστική δεν μπορεί να έχει αποκλειστεί από μια διένεξη συγχώνευσης.|
|Ακεραιότητα repo|    Αίτηση ελκυστική περιέχει εμφανή παλινδρομήσεις περιεχομένου.|
|Ακεραιότητα repo|    Αίτηση ελκυστική δεν περιλαμβάνει ένα ενσωματωμένο repo ή οποιαδήποτε ασυνήθιστο, περιττά αρχεία.|
|Ακεραιότητα repo |Αίτηση ελκυστική περιέχει λιγότερους από 100 αρχεία που τροποποιήθηκαν, εκτός εάν η ΤΙΜΉ σκόπιμα ενημερώνει υποκατάστημα έκδοση από το υπόδειγμα. (Πραγματικά, μια ΤΙΜΉ θα πρέπει να περιέχει άκρο μικρότερο από αυτά, αλλά μετά από 100 αρχεία η αλλαγή, GitHub δεν εμφανίζεται το diffs).|
|Ονομασία |Ονόματα αρχείων για νέα αρχεία ακολουθήστε το [αρχείο ονομασία κατευθυντήριες γραμμές](file-names-and-locations.md).|
|Ονομασία |Νέους φακέλους εισάγονται το repo, ακολουθήστε τις [οδηγίες ονομασίας φακέλου](file-names-and-locations.md#folder-names-in-the-repo).|
|Περιεχόμενο    |Το άρθρο είναι μια τεχνική έγγραφο, και, συνεπώς, στο σωστό περιεχόμενο κανάλι. Ανατρέξτε στο θέμα το [Τι μεταβαίνει πού καθοδήγηση](content-channel-guidance.md).|
|Περιεχόμενο    |Το θέμα στο έγγραφο τεχνική είναι κατάλληλη για ένα άρθρο τεχνική. Ανατρέξτε στο θέμα το [Τι μεταβαίνει πού καθοδήγηση](content-channel-guidance.md).|
|Περιεχόμενο    |Το άρθρο περιέχει μια εισαγωγική παράγραφο και μια σχετικά με τη διαδικασία ή εννοιολογική σώμα του περιεχομένου. Το άρθρο πρέπει να περιέχει επαρκή, ολοκλήρωσης περιεχόμενο σε αναμονή ξεχωριστή ως ένα άρθρο. Δεν θα πρέπει να είναι ένα μικρό τμήμα των πληροφοριών. (Εξαίρεση: ένα θέμα "Όρια" Εάν είναι στο περιβάλλον ενός μεγάλου άρθρου που παραθέτει όλα τα όρια της υπηρεσίας.)|
|Περιεχόμενο| Γίνεται αρίθμηση των στοιχείων που θα πρέπει να είναι οι λίστες με αρίθμηση, στοιχεία που πρέπει να είναι ακανόνιστα λίστες με κουκκίδες. Αυτό είναι το βασικό χρηστικότητας.|
|Περιεχόμενο| Ασυνήθιστα ή νέων γραφικά, αρχιτεκτονική πληροφοριών ή δομές ή εμφανώς μη τυπική σχέδια πρέπει να είναι vetted με ο αναθεωρητής ΤΙΜΉ υποψήφιου πελάτη. Ομάδες στις οποίες δοκιμές με νέες δυνατότητες πρέπει να έχετε ένα πρόβλημα/λύση καμβά ή ένα σχέδιο στο σημείο για την αξιολόγηση δοκιμές.|
|Λειτουργίες σχεδίαση/τοποθεσίας| Switchers χρησιμοποιούνται μόνο για την εναλλαγή μεταξύ πολλών εκδόσεων του ίδιου άρθρου.|
|Λειτουργίες σχεδίαση/τοποθεσίας| Οι τίτλοι των switchered άρθρα περιέχουν πληροφορίες που διαφοροποιεί κάθε άρθρο από τα άλλα άρθρα στο σύνολο switchered.|
|Λειτουργίες σχεδίαση/τοποθεσίας| Δημιουργήθηκε με μη αυτόματο τρόπο πίνακες περιεχομένων δεν επιτρέπονται. Το άρθρο πρέπει να βασίζεστε στη H2s για το πεδίο TOC στη σελίδα.|
|Λειτουργίες σχεδίαση/τοποθεσίας| Εάν υπάρχουν H2 επικεφαλίδες, το άρθρο περιέχει τουλάχιστον δύο H2 επικεφαλίδες. Χρησιμοποιώντας ένα επικεφαλίδα H2 δημιουργεί ένα άρθρο ενός στοιχείου πίνακα Περιεχομένων. Πρέπει να χρησιμοποιούνται επικεφαλίδες H2 πριν H3 επικεφαλίδες για να βεβαιωθείτε ότι έχει δημιουργηθεί από έναν πίνακα Περιεχομένων.|
|Markdown| HTML: Περιεχόμενο προέλευσης δεν περιέχει HTML στο επίπεδο μπλοκ – μικρή ενσωματωμένη επιτρέπεται HTML – όπως εκθέτης, δείκτης, ειδικών χαρακτήρων και άλλες δευτερεύουσες ενέργειες που δεν μπορείτε να κάνετε με το markdown. Οι πίνακες HTML επιτρέπονται ΜΌΝΟ εάν ο πίνακας περιέχει λίστες με κουκκίδες ή αρίθμηση, αλλά αυτό είναι συνήθως το περιεχόμενο που πρέπει να είναι απλοποιημένα, ώστε να μπορεί να έχει κώδικα, το αρχείο προέλευσης στο markdown ένδειξη.|
|Markdown   |Προσαρμοσμένη markdown στοιχεία χρησιμοποιούνται ανάλογα με την περίπτωση. Ex: Σημειώσεις κωδικοποιούνται χρησιμοποιώντας το AZURE. ΣΗΜΕΙΏΣΤΕ την επέκταση, δεν ως απλό κείμενο.|
|SEO    |Το "& #124- Απαιτείται Microsoft Azure"αναγνωριστικό τοποθεσίας|
|SEO    |Ο τίτλος Η1 περιέχει επαρκείς πληροφορίες για να περιγράψετε το περιεχόμενο αυτού του άρθρου, για να διαφοροποιήσετε από άλλα άρθρα Azure και να αντιστοιχίσετε σε λέξεις-κλειδιά πιθανό πελάτη. Για παράδειγμα, "Επισκόπηση" ως τίτλο Η1 είναι ένα πρόβλημα.|
|Ορολογία| Η χρήση της το ακρωνύμιο ARM, V1 ή V2 ως αναφορές για να το κλασικό και μοντέλα ανάπτυξης για τη διαχείριση πόρων είναι ένα στοιχείο ορολογία αποκλεισμού.|
|Εικόνες |Δεν γίνονται δεκτές αρχεία GIF με κίνηση σε το repo.|
|Εικόνες | Εικόνες έχουν Απαλοιφή ανάλυση, είναι δωρεάν λέξεις με ορθογραφικά λάθη και δεν περιέχουν ιδιωτική πληροφορίες | 
|Ενδιάμεσου σταδίου|Η προεπισκόπηση άρθρο πρέπει να είναι καθαρή σε ενδιάμεσου σταδίου. Δεν μπορεί να περιέχει εμφανή προβλήματα μορφοποίησης: <br><br>-Μια λίστα με κουκκίδες ή αρίθμηση που εμφανίζεται ως μιας παραγράφου <br> -Κωδικός στο μπλοκ κώδικα που εμφανίζονται εν μέρει στο μπλοκ κώδικα και εν μέρει εκτός <br>-Αρίθμηση βήματα εσφαλμένη αρίθμηση λόγω ελαττωματική εσοχές|

## <a name="non-blocking-content-quality-items"></a>Αποκλεισμός μη περιεχομένου με την ποιότητα στοιχείων

Για αυτά τα στοιχεία, οι αναθεωρητές αίτηση ελκυστική παρέχουν σχόλια και οδηγίες για το συντάκτη για παρακολούθηση θέματος με διορθωτικές ενέργειες σε ένα αίτημα ελκυστική νεότερη έκδοση. Ωστόσο, αυτή σχολίων δεν εμποδίζει τη δική σας απόφαση για τη συγχώνευση. Οι συντάκτες θα πρέπει να υπενθύμισης εντός 3 εργάσιμων ημερών με διορθωτικές ενέργειες.

| Κατηγορία | Ποιότητα αναθεώρηση στοιχείου |
|----------|---------------------|
|Περιεχόμενο|Άρθρα θα πρέπει να έχετε μια "επόμενα βήματα" στο τέλος με 1-3 σχετικές και συναρπαστικές επόμενα βήματα. Σύντομο κείμενο πρέπει να περιλαμβάνονται που βοηθά στον πελάτη Κατανόηση γιατί σχετίζονται τα επόμενα βήματα. (Νέα άρθρα μόνο). Παράδειγμα: <https://azure.microsoft.com/en-us/documentation/articles/xplat-cli-install/><br>![](media/contributor-guide-pr-criteria/nextstepsexample.PNG)|
|Περιεχόμενο|Ορθογραφικός έλεγχος, γραμματικός έλεγχος και άλλα θέματα συγγραφής - ελκυστική αίτηση αναθεωρητές μπορεί να σχολιάσετε μερικά θέματα μικρή ως μη-αποκλεισμό των σχολίων. Εάν υπάρχουν περισσότερα από μερικά θέματα σύνταξης, οι αναθεωρητές συνδεθείτε μιας αίτησης επεξεργασίας για το άρθρο για μια επεξεργασία μετά τη δημοσίευση.|
|Εικόνες|Χρησιμοποιήστε εικόνες σωστή επεξήγηση στυλ και το χρώμα και στιγμιότυπα οθόνης, χρησιμοποιήστε το σωστό στυλ περιγράμματος και κράτησης θέσης. [Ανατρέξτε στις οδηγίες την εικόνα](https://github.com/Azure/azure-content/blob/master/contributor-guide/create-images-markdown.md).|
|Εικόνες|Οι εικόνες περιέχουν εναλλακτικό κείμενο. [Ανατρέξτε στις οδηγίες την εικόνα](https://github.com/Azure/azure-content/blob/master/contributor-guide/create-images-markdown.md).|
|Λειτουργίες σχεδίαση/τοποθεσίας|Τις επικεφαλίδες H2, όταν απόδοσης του πίνακα περιεχομένων στη σελίδα, θα πρέπει να αναδιπλώνεται ιδανικά δεν υπερβαίνει τα 2 γραμμές. Περισσότερο οι επικεφαλίδες κάνουν το άρθρο TOC δυσκολότερη η σάρωση.|
|Συμβάσεις στυλ|Όλες οι τίτλοι και οι επικεφαλίδες είναι κεφαλαία-πεζά πρότασης, ανά Azure στυλ.|
|Διαδικασία|Εάν η πρόσκληση σε ελκυστική θα μπορούσε να να εύκολα δηλώσετε για να επωφεληθείτε από PRmerger αυτοματισμού, οι αναθεωρητές αίτηση ελκυστική παροχή σχολίων για το συντάκτη σχετικά με τη χρήση διακλαδώσεις, ώστε να ήταν δυνατή η συγχώνευση αυτόματα τις αλλαγές. Ανατρέξτε [στο άρθρο πρότυπα ΤΙΜΉ](https://github.com/Azure/azure-content/blob/master/contributor-guide/contributor-guide-pull-request-etiquette.md#in-a-hurry-submit-prs-that-can-be-accepted-automatically).|
|Διαδικασία|Όταν διαγράφετε ή μετονομασία ενός άρθρου, βεβαιωθείτε ότι μπορείτε να ακολουθήσετε τη διαδικασία. Ελκυστική αίτηση αναθεωρητές θα πρέπει να προσθέσετε την παρακάτω σχολίου και σύνδεσης σε ένα σχόλιο:<br><br>*Βεβαιωθείτε ότι έχετε ακολουθήσει τη διαδικασία οι συνεργάτες του οδηγού για τη διαγραφή άρθρα: <https://github.com/Azure/azure-content/blob/master/contributor-guide/retire-or-rename-an-article.md> .*|
